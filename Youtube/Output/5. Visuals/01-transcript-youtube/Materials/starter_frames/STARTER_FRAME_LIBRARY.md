# Starter Frame Library — 01-transcript-youtube

Use these frames as approved or draft first frames for Higgsfield clips. Every matching end frame should feel like it naturally emerges from the selected starter: keep the same palette, camera world, panel depth, and empty-space logic.

Before production use, normalize selected frames to `1920x1080`.

## Shared Rules

- Keep the starter frame clean: no main subject already completed in the center.
- Put exact readable text, charts, UI labels, and checklists in HyperFrames when accuracy matters.
- End frames should add one clear visual idea, not several competing ideas.
- For app/screenshot-style starters, preserve the app shell and place future visuals inside the empty work area.
- For abstract starters, build the end frame from the starter's existing edge language: routes, panels, nodes, glass panes, or terminal surfaces.

## End Frame Prompt Pattern

Use this when pairing a starter with a final image:

```text
Create a 3-second Higgsfield clip using [starter file] as the first frame and [end frame] as the final reference.

Start from the clean starter exactly: preserve its palette, camera angle, app shell/background, and empty center.

Animate one clear visual idea into the empty area: [describe the concept].
The final frame should match the end reference: [describe layout].

Motion: restrained, premium, smooth easing, gentle parallax only where it fits the starter.
Do not add unrelated UI, extra text, logos, humans, devices, glitch effects, harsh neon, or busy dashboard noise.
If exact text is needed, leave room for HyperFrames text and do not rely on Higgsfield for readability.

Duration: 3 seconds. Aspect ratio: 16:9.
```
