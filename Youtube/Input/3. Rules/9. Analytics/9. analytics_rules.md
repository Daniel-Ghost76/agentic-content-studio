# Analytics Rules

Claude owns Stage 9 by default.

Standards for Stage 9. Every analytics action must comply.

---

## Data Integrity

**R1 — Never manually edit snapshot files.**
Snapshot files are written once by `fetch_analytics.py` pulling from the YouTube API. Never overwrite or adjust numbers by hand. If a snapshot is wrong, delete the file and re-fetch.

**R2 — Never create snapshot files without API data.**
Do not generate placeholders, estimates, or manually filled snapshots. All numbers come from the API or the file does not exist.

**R3 — Never re-fetch an existing snapshot.**
If `{project_id}_analytics_d07.md` already exists in `Youtube/Output/9. Analytics/{project_id}/`, do not overwrite it. The Day 7 window is a fixed point in time — re-fetching later would pull different numbers and make the snapshot meaningless.

---

## Snapshot Timing

**R4 — Day 7 snapshot uses data from publish date through publish date + 7 days.**
The end date is capped at today minus 2 days (YouTube's data delay). Do not extend the window beyond the 7-day boundary.

**R5 — Day 30 snapshot uses data from publish date through publish date + 30 days.**
Same delay cap applies.

**R6 — Never fetch a snapshot before the window is available.**
If today minus the publish date is less than the snapshot window plus the 2-day data delay, skip and log — do not error.

---

## Monthly Rollup

**R7 — Monthly rollup is generated once, in the last week of the month.**
Do not regenerate if `YYYY-MM-monthly.md` already exists. If it needs updating (e.g., a late snapshot came in), delete the file and regenerate cleanly.

**R8 — Monthly rollup only includes videos with at least one snapshot.**
Do not include videos that have no Day 7 or Day 30 data yet. Note their absence in the rollup.

---

## Logging

**R9 — Every `/analytics` run must log what it did.**
Log: videos checked, snapshots due, snapshots fetched, snapshots skipped (with reason), monthly rollup status. Silence is not acceptable.

**R10 — upload_log.md is the source of truth for published videos.**
Never look for published videos anywhere else. Each row must include `project_id`. If a video was uploaded but not in the log, it should not be tracked.

---

## Interpretation

**R11 — Flag underperforming videos, do not ignore them.**
If a Day 7 CTR is below 4% or avg view duration is below 40%, note it explicitly in the snapshot or as a comment to the user. These are actionable signals.

**R12 — Do not draw conclusions from Day 7 data alone.**
Day 7 is an early signal, not a verdict. Wait for Day 30 before making strong judgements about a video's performance.
