# Pre-production Materials Rules (Stage 3)

Codex owns and executes Stage 3.

## Stage Boundary

- Stage 3 happens before recording.
- Stage 3 creates slides, diagrams, screen-share visuals, and static recording assets.
- Stage 3 does not edit raw footage or composite overlays onto cut footage.
- Stage 5 handles post-production visuals and overlays.

## Source Of Truth

- The approved Visuals Brief is the main source of truth.
- The Visual Production Library defines reusable visual styles and prompt capsules:
  `Youtube/Input/4. Resources/0. visual_production_library/`
- The approved script provides context if the brief is ambiguous.
- Daniel's latest instruction overrides older briefs or style references.

## Output Rules

- Save assets to `Youtube/Output/3. Pre-production Materials/{project_id}/`.
- Every artifact must start with `{project_id}_`.
- Include a `{project_id}_production_note.md` with source brief, dimensions, intended use, chosen visual library capsule, and any timing notes.
- Do not overwrite existing production assets unless Daniel explicitly asks.

## Text And Layout

- Preserve exact on-screen text from the brief.
- Use readable type and generous spacing.
- Avoid clutter. The material should help Daniel record, not compete with him.
- Verify image dimensions for all exported slides and visual assets.

## Ownership

- Codex executes this stage directly.
- Claude hands off to Codex by producing the Stage 3 Visuals Brief from scripting.
