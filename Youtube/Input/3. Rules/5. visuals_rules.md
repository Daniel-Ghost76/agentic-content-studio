# Visuals Rules (Stage 5)

Codex owns and executes Stage 5.

## Stage Boundary

- Stage 5 happens after Stage 4 editing is complete.
- Stage 5 starts from `{project_id}_cut_final.mp4` (graded + zoom-baked) and `{project_id}_overlay_map.md` (approved by Daniel).
- Stage 5 creates post-production overlays, motion graphics, visual callouts, full-screen Higgsfield clips, intro/outro, and does final QC.
- Stage 5 does not create pre-recording slides or presentation materials. Those belong to Stage 3.
- Stage 5 does not make new overlay decisions. All overlay decisions are made in Stage 4c (Overlay Identifier). If the map is missing something, flag it back to Daniel — do not add overlays unilaterally.

## Higgsfield Rules

- **Credit cap is set in Stage 4c (Overlay Identifier) setup.** Agent 5a cannot exceed this cap without Daniel confirmation.
- **Check balance before generation.** Use `mcp__claude_ai_Higgsfield__balance` before starting any Higgsfield generation. If balance is insufficient, pause and ask Daniel which slots to prioritize.
- **Review each clip before compositing.** Never composite a Higgsfield clip without visual review. Flag if the generation looks wrong.
- **Each full-screen clip = 1 credit.** Track usage. Report remaining balance at end of session.
- **Default cap: 5 clips per video** unless Daniel specifies otherwise at Stage 4c setup.

## Flag-and-Confirm Rule

Stage 5 agents are allowed to make judgment calls when the map is ambiguous or a placement causes a problem. When doing so:
1. **Never adjust silently and never block.** Always flag explicitly.
2. State what you did or plan to do, and why.
3. Wait for Daniel's confirmation before compositing the affected slot.
4. Document all adjustments in `{project_id}_overlay_check.md`.

## Output Rules

- Screen recording cleanup → `screen_rec/{project_id}_screen_rec_clean.mp4`
- Animation slots → `animations/slot_NNN/{project_id}_overlay_slot_NNN.mp4`
- Higgsfield clips → `higgsfield/slot_NNN_clip.mp4`
- Composited video → `{project_id}_overlaid.mp4`
- Final master → `{project_id}_final.mp4` (after intro/outro/music/QC)
- QC report → `{project_id}_qc_report.md`
- Verify all output specs: 1920×1080, H.264, AAC 48kHz, 30fps.
- Each overlay check must name the Visual Production Library capsule used for each visual slot.

## Visual Production Library

- Use `Youtube/Input/4. Resources/0. visual_production_library/` before creating overlays, full-screen visuals, Higgsfield clips, emphasis word overlays, or CTA visuals.
- Partial overlays and full-screen overlays are separate library categories.
- If no approved capsule fits, ask Daniel whether to build a new draft capsule instead of inventing an undocumented style.

## Overlay Technical Rules

- **Subtitles applied LAST** — after all overlays in the filter chain.
- **All overlays use `setpts=PTS-STARTPTS+T/TB`** — correct PTS offset.
- **Parallel sub-agents for multiple overlay slots** — never sequential.
- **HyperFrames: lint and validate must pass** before rendering.
- **Overlay timestamp drift tolerance:** ±2 frames (±66ms at 30fps).
- **Duration floor:** ≥ 3s for simple cards, ≥ 5–7s for data/step content.
- **Hold final frame ≥ 1s** before the cut following any overlay.
- **Easing:** never `linear`. Use `ease_out_cubic` (single reveals) or `ease_in_out_cubic` (continuous draws).

## Brand Rules

- Intro/outro are cached by brand version in `brand_assets/`. Never re-render if a matching cached version exists.
- Never delete old brand versions — increment the version number for refreshes.
- No SVG logo in video compositions — text wordmarks only (logo not finalized).
- CTA in outro must include Skool community link.
