# Publishing Rules

Claude owns Stage 7 by default. Codex may orchestrate mobile publishing when Daniel asks from the Codex app.

Standards and constraints for Stage 7. Every upload must comply.

---

## Upload Rules

**R1 — Always confirm before uploading.**
Show the ranked order, schedule assignments, and metadata summary to the user. Get explicit confirmation before the upload script runs.

**R2 — Never publish immediately.**
All uploads must use `--schedule-at`. There are no exceptions. The upload script sets the video as private with an auto-publish date — it will never go live the moment it uploads.

**R3 — Never re-upload a video in `published/`.**
If a video is in `Youtube/Output/7. Publishing/{project_id}/published/`, it is done. Do not upload it again. If a genuine re-upload is needed, Daniel must explicitly ask.

**R4 — A canonical companion `.yaml` is required before upload.**
The required pair is `{project_id}_publish.mp4` plus `{project_id}_metadata.yaml` inside `Youtube/Output/7. Publishing/{project_id}/`. Never invent and silently use metadata. If YAML is missing, draft one and get user approval first.

**R5 — Title must be ≤ 100 characters.**
YouTube enforces this limit hard. Warn and stop if exceeded.

**R6 — Description must be ≤ 5,000 characters.**
YouTube enforces this limit hard. Warn before truncating; truncate only with user approval.

---

## Scheduling Rules

**R7 — Timezone is always Europe/London.**
Use `Youtube/Input/4. Resources/7. publishing_resources/schedule_slots.py` to compute slot datetimes. Never hardcode UTC offsets — BST (+01:00) and GMT (+00:00) differ by season and the script handles this automatically.

**R8 — Slot assignment order: first 3 Fridays, then Tuesdays.**
Videos ranked 1–3 go to the next 3 Friday 4:00 PM slots. Videos ranked 4+ go to consecutive Tuesday 4:00 PM slots. No exceptions.

**R9 — Ranked order is preserved.**
A lower-ranked video must never publish before a higher-ranked video. Slot 1 always goes to the highest-ranked video.

**R10 — Schedule time must be at least 5 minutes in the future.**
The YouTube API rejects `publishAt` values that are too close to now. `schedule_slots.py` always produces future dates so this is automatically satisfied.

---

## Metadata Quality Rules

**R11 — Description must open with the hook.**
The first 150 characters appear in search results and email notifications. The compelling hook goes here — not the channel name or a generic "In this video."

**R12 — Add the Skool community link to descriptions once it exists.**
When the Skool community is live, add the link to the links section of every description. Until then, omit it — descriptions can be edited in YouTube Studio after publishing.

**R13 — `made_for_kids` is always `false`.**
This channel produces content for adult professionals. Never set this to `true`.

---

## File Management Rules

**R14 — Only move files after a confirmed successful upload.**
Confirmed = the YouTube API returned a video ID. Do not move files on error or partial upload.

**R15 — Always log every upload to the Analytics log.**
Every successful upload gets an entry in `Youtube/Output/9. Analytics/upload_log.md` with `project_id`, date, title, video ID, URL, and scheduled time.

---

## Credentials Rules

**R16 — Never commit or share `credentials.json` or `token.json`.**
These contain private OAuth credentials. They must not appear in any shared file, Skool post, or version control system.

**R17 — If `token.json` is invalid, delete and re-auth cleanly.**
Do not patch or manually edit token files. Delete `token.json` and run `/publish` — the OAuth browser flow will regenerate it.

---

## Scope Rule

**R18 — Stage 7 ends at the YouTube scheduled URL.**
Once the URL and schedule time are confirmed and logged, this stage is complete. Cross-platform distribution is Stage 8.
