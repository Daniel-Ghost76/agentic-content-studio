# Editing Rules

Claude owns Stage 4 by default.

Rules for all Stage 4 agents: 4a (Prep), 4b (Cut Edit), 4c (Overlay Identifier), 4d (Grade + Zoom). Hard rules are correctness — deviation causes silent failures. Everything else is taste.

---

## Hard Rules (non-negotiable)

**ABSOLUTE: Never delete raw recorded footage.** Files inside `Youtube/Output/4. Editing/` that are raw or source recordings must never be deleted under any circumstances. This covers:
- Everything inside `originals/` (untouched camera exports)
- `{project_id}_raw.mp4` (normalized working source)
- Any other imported or recorded source file

Deleting a source file is irreversible. No workflow step, cleanup pass, or disk-space concern overrides this rule. The only valid override is Daniel explicitly asking to delete a specific file in that session.

**Never touch original imports.** Files in `Youtube/Output/4. Editing/{project_id}/originals/` are immutable. Normalized working copies and outputs stay inside `Youtube/Output/4. Editing/{project_id}/`.

**Confirm strategy before executing.** Never cut or render until Daniel has approved the plain-English strategy. This applies to both stages.

**One file at a time.** One raw video → one cut edit session. One cut video → one overlay session. Do not batch unless explicitly instructed.

**30ms audio crossfades at every cut boundary.** `afade=t=in:st=0:d=0.03,afade=t=out:st={dur-0.03}:d=0.03`. Hard pop on every cut that skips this.

**Per-segment extract → lossless concat** (not single-pass filtergraph). Double-encoding silently degrades quality when overlays are added later.

**Never cut inside a word.** Snap every cut edge to a word boundary from the transcript. Use padding (50–80ms) to absorb timestamp drift.

**Subtitles are applied LAST.** After every overlay in the filter chain. Overlays on top of burnt subtitles hide the text — silent failure.

**Overlay PTS shift.** Always use `setpts=PTS-STARTPTS+T/TB` for overlays so the animation's frame 0 aligns to its window start.

**Cache transcripts.** Never re-transcribe a source that already has `{project_id}_transcript.json`. Transcription is slow — the cache is the contract.

**Output spec.** All outputs must be: 1920×1080, H.264, AAC 48kHz, 30fps. Verify with `ffprobe` before confirming delivery.

## Prep Rules (Stage 4a)

**Never modify `originals/`** — normalized copies go to `normalized/`. Raw files are forever immutable.

**Sync fallback is mandatory.** If audio cross-correlation fails or screen recording has no compatible audio, stop and ask Daniel for a manual timestamp. Never guess the sync offset.

**DeepFilterNet for audio enhancement.** Do not fall back to ffmpeg `anlmdn` without flagging it to Daniel. DeepFilterNet is the required default.

**Multi-source EDL format.** When multiple sources exist, the EDL `sources` dict must map slug names to `normalized/` paths (not `originals/`). Primary transcription source is `audio/{project_id}_audio_enhanced.mp4`.

## Cut Edit Rules

**Filler word list (fixed unless overridden by Daniel):**  
`um`, `uh`, `ah`, `like` (when used as filler, not as a verb), `you know`, `so` (at sentence starts), `basically`

**Silence thresholds:**  
- Remove gaps ≥ 500ms between sentences  
- Preserve gaps < 300ms (natural speech cadence — removing these sounds robotic)  
- Gaps 300–500ms: judgment call based on pacing

**Cut padding working window:** 30–200ms. Default 50ms before, 80ms after. Tighter for fast-paced content, looser for cinematic.

**Prefer silences ≥ 400ms as cut targets.** Gaps 150–400ms are usable with a visual check. < 150ms is unsafe (mid-phrase risk).

**Self-eval before delivery.** `timeline_view.py` on the rendered output at every cut boundary. 3-pass cap — flag remaining issues rather than looping forever.

## Overlay Identifier Rules (Stage 4c)

**Stubs only.** The overlay identifier writes brief stubs — not complete HyperFrames specs. Complete specs are written by Agent 5 at build time.

**Higgsfield cap is a hard limit.** The cap is set at Agent 3c setup. Cannot be exceeded without Daniel confirmation.

**Daniel approves the overlay map before Agent 5 starts.** No building until the map is confirmed.

**Zoom budget:** Maximum 1 zoom per 60 seconds of cut video duration.

**Zoom peak limits:**
- Subtle: 1.05–1.08x
- Noticeable: 1.08–1.15x
- Never exceed 1.20x without Daniel confirmation.

---

## Grade + Zoom Rules (Stage 4d)

**One render pass only.** Grade and zoom are applied together. No double-encoding.

**Zoom before overlays.** Zoom is baked into `cut_final.mp4`. Overlays composite on top. Never zoom after overlays — the overlay text would zoom too.

**Zoom easing is never linear.** Always `ease_in_out_cubic`. Linear zoom looks mechanical.

---

## Overlay Rules

**Visual library categories.** Stage 4 does not build visual assets, but any overlay map or project note that identifies visual opportunities should use the Visual Production Library categories from `Youtube/Input/4. Resources/0. visual_production_library/selection_matrix.md`.

**HyperFrames: lint and validate must pass** before rendering. `npx hyperframes lint` + `npx hyperframes validate`. These catch structural and runtime errors early.

**Overlay timestamp drift tolerance:** ±2 frames (±66ms at 30fps). Anything wider is visible misalignment.

**Never burn subtitles before compositing overlays.** Subtitles are a final-step decision, not part of the base video.

**Parallel sub-agents for multiple overlays.** Never sequential. Use the `Agent` tool to spawn N at once.

**Duration floor for readable overlays:** ≥ 3s for simple cards, ≥ 5–7s for data or step content. Below this the viewer can't parse it.

**Hold the final frame ≥ 1s** before the cut following any overlay.

**Animation easing:** never `linear`. Use `ease_out_cubic` for single reveals, `ease_in_out_cubic` for continuous draws.

**Style confirmation before build.** If Daniel has not specified a palette and font, propose one and wait for approval. Never assume a default style.
