# Grade + Zoom Skill — How to Grade and Bake Zoom into the Base Video

Claude executes this stage. One render pass: grade applied, zoom baked in. Follow these steps in order.

---

## Step 1 — Run Setup Questions

Ask Daniel the three setup questions in `4d. grade_zoom_sub-agent.md`. Record answers.

---

## Step 2 — Read Zoom Keyframes from Overlay Map

Open `{project_id}_overlay_map.md` and extract all `[ZOOM]` entries. Build a zoom schedule:

```python
zoom_schedule = [
    {"timestamp": 30.8, "peak": 1.10, "ease_in": 0.8, "hold": 1.2, "ease_out": 0.7},
    {"timestamp": 135.3, "peak": 1.08, "ease_in": 0.6, "hold": 1.0, "ease_out": 0.5},
]
```

If no zoom entries exist, skip zoom step entirely.

---

## Step 3 — Analyze Grade

Run `grade.py` in analyze mode on each segment in `clips_graded/`:

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/grade.py" \
  --analyze "Youtube/Output/4. Editing/{project_id}/clips_graded/"
```

Review output: check for outlier segments (significantly different brightness, color temperature, saturation vs the majority). Flag any outliers to Daniel before proceeding.

Generate before/after sample frames for the first and last segment:

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/timeline_view.py" \
  "Youtube/Output/4. Editing/{project_id}/clips_graded/seg_001.mp4" 0 3
```

Write `grade_review/seg_001_before.png` from the sample. Apply grade and generate `seg_001_after.png`.

---

## Step 4 — Confirm Grade Direction

Based on setup question answer and analysis:

| Preset | When to use |
|--------|-------------|
| `auto` | Default — per-segment analysis, consistent look |
| `warm_cinematic` | Talking head with warm lighting, slightly desaturated blues |
| `neutral_punch` | Screen recording heavy, needs contrast without color shift |
| `none` | Footage is already graded or Daniel explicitly skips |

Write `{project_id}_grade.json`:
```json
{
  "preset": "auto",
  "outliers": [],
  "per_segment_overrides": {}
}
```

Update `{project_id}_edl.json` `grade` field to match.

---

## Step 5 — Build Zoom Filter

For each zoom keyframe, build an ffmpeg `zoompan` filter expression. The zoompan filter operates on the output timeline (post-cut timestamps).

```python
# Pseudocode — generate ffmpeg zoompan expression
# For each zoom keyframe:
#   ease_in phase: zoom from 1.0 to peak over ease_in seconds
#   hold phase: maintain peak zoom for hold seconds  
#   ease_out phase: zoom from peak back to 1.0 over ease_out seconds
# Between keyframes: zoom = 1.0 (no zoom)
# Use cubic easing: z = start + (end-start) * (3t² - 2t³) where t = phase_progress
```

The actual ffmpeg zoompan filter string will be written by `render.py` when the EDL includes zoom entries. Add zoom entries to `{project_id}_edl.json`:

```json
"zooms": [
  {
    "output_timestamp": 30.8,
    "peak": 1.10,
    "ease_in_s": 0.8,
    "hold_s": 1.2,
    "ease_out_s": 0.7
  }
]
```

---

## Step 6 — Single Render Pass

Run render with both grade and zoom:

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/render.py" \
  "Youtube/Output/4. Editing/{project_id}/{project_id}_edl.json" \
  -o "Youtube/Output/4. Editing/{project_id}/{project_id}_cut_final.mp4" \
  --apply-grade \
  --apply-zoom
```

This is one pass. Do not run grade and zoom separately — double-encoding degrades quality.

---

## Step 7 — Verify Output

```bash
ffprobe -v quiet -print_format json -show_streams \
  "Youtube/Output/4. Editing/{project_id}/{project_id}_cut_final.mp4"
```

Confirm: 1920×1080, H.264, AAC 48kHz, 30fps, duration matches `{project_id}_cut.mp4` ±0.5s.

Sample zoom moments using `timeline_view.py`:
- For each zoom keyframe, sample at: ease-in start, peak, ease-out end
- Verify smooth push-in, no jitter, easing looks cinematic not mechanical

---
