# Cut Edit Skill — How to Cut a Raw Video

Claude executes this stage. One video in, one clean cut out. Follow these steps in order.

---

## Step 1 — Run Setup Questions

Ask Daniel the four setup questions in `4b. cut_edit_sub-agent.md` before doing any work. Record answers.

---

## Step 2 — Locate and Inspect Source

Primary source is the audio-enhanced talking head:
```bash
ffprobe -v quiet -print_format json -show_streams \
  "Youtube/Output/4. Editing/{project_id}/audio/{project_id}_audio_enhanced.mp4"
```

If Agent 1 (Prep) has not yet run and `audio_enhanced.mp4` does not exist, fall back to:
```bash
ffprobe -v quiet -print_format json -show_streams \
  "Youtube/Output/4. Editing/{project_id}/normalized/{project_id}_talking_head.mp4"
```

Note duration, resolution, fps, audio codec. Set:
- `<edit_dir>` = `Youtube/Output/4. Editing/{project_id}/`
- `<source>` = path to audio_enhanced.mp4 (or fallback)

Read `{project_id}_manifest.json` to know all sources and their slugs.

---

## Step 3 — Transcribe with ElevenLabs Scribe

Cached — skips if transcript already exists.

```bash
export ELEVENLABS_API_KEY=$(grep '^ELEVENLABS_API_KEY=' ~/.claude/.env | cut -d'=' -f2-)
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/transcribe.py" \
  "<source>" \
  --edit-dir "<edit_dir>" \
  --language en
```

Output: `<edit_dir>/{project_id}_transcript.json` — word-level timestamps.

---

## Step 4 — Pack Transcript

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/pack_transcripts.py" \
  --edit-dir "<edit_dir>"
```

Output: `<edit_dir>/{project_id}_takes_packed.md` — phrase-level view with `[start-end]` timestamps.

---

## Step 5 — Pre-scan (Required)

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/validate_filler_words.py" \
  "<edit_dir>/{project_id}_transcript.json"
```

Use printed output. Do not manually scan. Note verbal slips, filler-heavy passages, long silences.

---

## Step 6 — Propose Cut Strategy

4–6 sentences covering:
- What gets removed (fillers, silences, bad takes, specific sections)
- Any takes or beats being prioritised
- Estimated output duration
- Anything flagged in pre-scan that needs special handling

**Wait for Daniel's approval before doing anything else.**

---

## Step 7 — Drill Ambiguous Moments

For any uncertain cut point:
```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/timeline_view.py" \
  "<source>" <start_seconds> <end_seconds>
```

Generates a filmstrip + waveform PNG. Use at decision points only — not as a scan tool.

---

## Step 8 — Build the EDL

Write `<edit_dir>/{project_id}_edl.json`. Multi-source format:

```json
{
  "version": 1,
  "sources": {
    "talking_head": "/abs/path/to/audio/{project_id}_audio_enhanced.mp4",
    "screen_rec": "/abs/path/to/normalized/{project_id}_screen_rec.mp4"
  },
  "ranges": [
    {
      "source": "talking_head",
      "start": 2.42,
      "end": 31.80,
      "beat": "HOOK",
      "quote": "first few words...",
      "reason": "Clean take, strong opening"
    }
  ],
  "grade": "none",
  "overlays": [],
  "subtitles": null,
  "total_duration_s": 0
}
```

Rules for every range:
- Use approximate timestamps from `takes_packed.md` — Step 8b will snap them to exact word boundaries
- Prefer silence gaps ≥400ms as the target cut point (Step 8b compresses anything left inside)
- Include `beat`, `quote`, `reason`

Set `total_duration_s` to the sum of all `(end - start)` values — `compress_gaps.py` (Step 8b) will recalculate this automatically.

---

## Step 8b — Compress Gaps (Required — run before render)

Fixes two mechanical problems: mid-word cut edges and long internal silences. Runs after the EDL is written; operates on the same EDL file in-place.

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/compress_gaps.py" \
  "<edit_dir>/{project_id}_edl.json" \
  --in-place
```

The script:
- Snaps every `start`/`end` to the nearest word boundary in the Scribe transcript — no more mid-word cuts
- Walks each kept segment word-by-word; any gap ≥ 400ms is split and compressed to 150ms total (75ms padding each side)
- Prints how many seconds were removed and the reduction percentage
- Updates `total_duration_s` in the EDL

Transcripts are auto-detected from `<edit_dir>/transcripts/`. Requires `transcribe.py` to have already run (Step 3).

**Tuning thresholds** (optional):
```bash
# More aggressive — cut gaps >= 300ms, keep only 100ms of silence
python3 "..." --max-gap 300 --keep-gap 100

# Conservative — only cut gaps >= 600ms
python3 "..." --max-gap 600 --keep-gap 200
```

Default (400ms / 150ms) is the right starting point for talking-head footage.

---

## Step 9 — Render the Cut

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/render.py" \
  "<edit_dir>/{project_id}_edl.json" \
  -o "Youtube/Output/4. Editing/{project_id}/{project_id}_cut.mp4"
```

Render: per-segment extract with 30ms audio fades → lossless concat.

---

## Step 10 — Self-Evaluate

Run `timeline_view.py` on the **rendered output** at every cut boundary (±1.5s):
- No visual flash or jump
- No waveform spike (audio pop)

Sample: first 2s, last 2s, 2–3 mid-points.

```bash
ffprobe -v quiet -print_format json -show_streams \
  "Youtube/Output/4. Editing/{project_id}/{project_id}_cut.mp4"
```

Verify duration matches `total_duration_s` ±1s. Cap self-eval at 3 passes.

---

## Step 11 — Persist Session Memory

Append to `<edit_dir>/{project_id}_project.md`:

```markdown
## Session N — Cut Edit — YYYY-MM-DD

**Strategy:** one paragraph
**Decisions:** take choices, cuts, and why
**Visual opportunities:** obvious moments for library-backed overlays, emphasis words, CTA visuals, proof screens, or full-screen breaks; final mapping belongs to Agent 3 / Stage 4c
**Reasoning log:** one-line rationale for non-obvious decisions
**Outstanding:** deferred items
```

Confirm location of `{project_id}_cut.mp4` to Daniel. Agent 3 (Overlay Identifier) is now ready to run.
