# Ideation Skill — How to Build One Ideation Document

Claude executes this stage by default.

One idea in, one PDF out. Follow these steps in order.

---

## Step 1 — Identify the Idea
Read the idea from `Youtube/Quick Access/youtube_ideas_30_order.pdf`.
Note the position number, folder title, and expanded idea.

---

## Step 2 — Live Research (Perplexity)
Before committing to the idea, run the live-web research tool. Claude's knowledge has a cutoff; this is the only step that sees the current web with citations.

```
python3 "Youtube/Input/5. Tools/1. ideation_tools/perplexity_research.py" --saturation "<idea topic>"
```

Read the result and judge: is this topic over-covered right now, and what is the freshest angle? Carry the recommended angle into The One Thing and Hook. Then pull hard specifics for the Specificity Bank:

```
python3 "Youtube/Input/5. Tools/1. ideation_tools/perplexity_research.py" --stats "<idea topic>"
```

(If the tool errors with a missing key, note it to Daniel and fall back to web search — do not block.)

---

## Step 3 — Find Reference Videos
Read the priority channels list at:
`Youtube/Input/4. Resources/1. ideation_resources/priority_reference_channels.md`

Search for the 2-3 most relevant videos on this exact topic. Work down the priority list first. Broaden only if nothing relevant is found on those channels. For a fast recent scan you can also run `--refs "<topic>"` on the research tool above.

For each video include:
- Title as a hyperlink
- Channel name
- One line: what to notice when watching it

Verify every URL is a real, accessible YouTube video before including it.

## Step 4 — Fill All Sections

### Frontmatter
```
title: [video title]
video no: [N of 30]
series: Building an AI Business From Scratch
voice_reference: ../../Input/4. Resources/2. scripting_resources/voice_reference/nate_herk_voice_reference.md
video_type: concept | tutorial | story
hook_type: Curiosity-Gap | Problem-Statement | Demonstration | Bold-Claim
script_versions: A only | B only | A + B
status: DRAFT
```

### Reference Videos
2-3 verified YouTube links from priority channels. Per video: title as hyperlink, channel name, one line on what to notice.

### The One Thing
One sentence. The single insight this video delivers. Everything else serves it.

### Hook
The exact hook line — not options, one decided line. Under 30 words.

### Target Viewer
One sentence, specific. Not "people interested in AI."
Example: "Someone who uses ChatGPT daily but can't explain what it actually does."

### Payoff
What can the viewer do or say after watching that they couldn't before? Work backwards from the ending, not forwards from the premise.

### Beat Structure
For each beat:
- **Beat — [Name]**
- **Point**: one sentence — what this beat establishes
- **Visual**: specific — what tool, chart, demo, or graphic is on screen
- **Target words**: ~Xw

### Analogy Bank
Minimum 2. One per major concept. Labeled by concept.

### Thesis Line Location
Which beat does the most important moment land in, and what is the core insight? (Scripting writes the exact line — ideation decides where and what.)

### Framework Phrase
3 words or fewer. The wrap. Decided here, not in scripting.
Example: "Predict, not retrieve."

### Specificity Bank
Real numbers, real tool names, real data points to use in this video. Scripting should not invent these.

### Constraints
What NOT to explain. Scope limits. Vocabulary to avoid.

### CTA Direction
What video does this tee up? One-line tease for the next video.

### Personal Angle [DANIEL TO FILL]
A real moment, outcome, or realization from Daniel's own experience that connects to this topic. Suggest a placeholder if Daniel hasn't filled it yet.

### Vulnerability Moment [DANIEL TO FILL]
One moment where Daniel admits a wrong assumption or mistake related to this topic. Note which beat it belongs in. Suggest a placeholder if Daniel hasn't filled it yet.

## Step 5 — Save as PDF
Save to: `Youtube/Output/1. Ideation/`
Filename: `[N]. [Folder_Title].pdf`

Use the exact folder title from the ordered list, with underscores for spaces.
Examples: `2. LLMs_Explained.pdf`, `5. Claude_Code_Setup.pdf`, `14. API_CLI_MCP.pdf`

Confirm the saved location to the user.
