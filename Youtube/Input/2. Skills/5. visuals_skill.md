# Visuals Skill (Stage 5)

Codex executes this stage.

This skill defines the post-production visuals workflow: overlays, motion graphics, composited callouts, and visual polish applied after Stage 4 editing.

For slides and other pre-recording assets, use Stage 3 instead.

## Core Visual Direction

Default post-production visual language is a soft premium AI-tech HUD style:

- dark teal / black interface environment
- low-to-medium contrast
- frosted glass panels
- muted cyan-teal glow
- visible dotted HUD texture
- soft blurred UI/code/dashboard depth
- large readable text when a phrase matters

Use the approved references in:

```text
Youtube/Input/4. Resources/5. visuals_resources/approved visuals/
```

## Stage 5 Workflow

1. Read the cut edit from `Youtube/Output/4. Editing/{project_id}/{project_id}_cut.mp4`.
2. Read the edit session files in `Youtube/Output/4. Editing/{project_id}/`.
3. Identify moments where overlays or visual polish help comprehension.
4. Confirm overlay timing, content, and style with Daniel when the overlay brief is not already locked.
5. Use Stage 5a overlay compositing for rendered motion graphics.
6. Save final overlaid videos to `Youtube/Output/5. Visuals/{project_id}/{project_id}_overlaid.mp4`.
7. Verify the output with `ffprobe` and sample-frame checks.

## Visual Rules

- Keep overlays readable and subordinate to Daniel or the screen recording.
- Do not cover face-cam or key UI unless Daniel explicitly approves.
- Avoid harsh neon, explosions, heavy glitch effects, and busy dashboards.
- Prefer smooth slide-in, soft scale-in, gentle parallax, tiny data particles, and restrained glow pulses.
- No logos, humans, or real objects unless explicitly requested.

## Relationship To Stage 3

Stage 3 creates materials before recording: slides, diagrams, and static recording assets.

Stage 5 creates visuals after recording: overlays, composited motion graphics, and final visual polish.
