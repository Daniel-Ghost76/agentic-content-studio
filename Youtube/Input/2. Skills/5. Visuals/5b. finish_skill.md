# Finish Skill — How to Add Intro/Outro, Music, and Run QC

Codex executes this stage. Follow these steps in order. QC always runs at the end, even if intro/outro and music are skipped.

---

## Step 1 — Run Setup Questions

Ask Daniel the four setup questions in `5b. finish_sub-agent.md`. Record all answers before touching any file.

---

## Step 2 — Inspect Overlaid Video

```bash
ffprobe -v quiet -print_format json -show_streams \
  "Youtube/Output/5. Visuals/{project_id}/{project_id}_overlaid.mp4"
```

Note: duration, resolution, fps, codec, audio. If spec is wrong, stop and flag before proceeding.

## Step 3 — Intro/Outro (if requested)

### Check Cache First

```bash
ls "Youtube/Input/4. Resources/5. visuals_resources/brand_assets/"
```

For each cached version, read its `spec.json` to check brand version ID. If a matching version exists, copy or symlink to episode folders:

```bash
cp "brand_assets/intro_v1.mp4" "Youtube/Output/5. Visuals/{project_id}/intro/{project_id}_intro.mp4"
cp "brand_assets/outro_v1.mp4" "Youtube/Output/5. Visuals/{project_id}/outro/{project_id}_outro.mp4"
```

Verify cached files with `ffprobe` — confirm spec before using.

### Build New Intro/Outro (if no cache or brand refresh requested)

Use Remotion. The existing `remotion-visuals/` project has glass panel compositions that are a strong starting point.

**Intro spec:** 5–8 seconds, 1920×1080, 30fps, H.264.
- Channel name / wordmark
- Episode title or number (if applicable)
- HUD animation (use existing glassPanel composition or build new)
- Soft teal/dark aesthetic matching brand guide

**Outro spec:** 10–15 seconds, 1920×1080, 30fps, H.264.
- Subscribe CTA (text, not logo — logo not finalized)
- Link to Skool community
- End-screen safe zones (YouTube reserves bottom-20% for end screen elements)

```bash
# From remotion-visuals/ directory:
npx remotion render IntroComposition "brand_assets/intro_v1.mp4"
npx remotion render OutroComposition "brand_assets/outro_v1.mp4"
```

Write `brand_assets/intro_v1_spec.json`:
```json
{"version": 1, "duration_s": 7, "resolution": "1920x1080", "fps": 30, "built": "YYYY-MM-DD"}
```

Copy to episode folders after caching.

## Step 4 — Music Integration (if requested)

Read `music_library.md` and select a track, or use Daniel's choice from setup.

**Ducking using transcript speech timing:**

```bash
# Extract speech regions from transcript as ffmpeg volume automation
python3 -c "
import json
with open('{project_id}_transcript.json') as f:
    t = json.load(f)
# Generate volume automation string: -18dB during speech, -6dB during silence
# Output: ffmpeg volume filter string with timeline entries
"

ffmpeg -i '{project_id}_overlaid.mp4' -i 'music/track.mp3' \
  -filter_complex '[1:a]volume=<automation_string>[music];[0:a][music]amix=inputs=2:weights=1 0.3[aout]' \
  -map 0:v -map '[aout]' \
  -c:v copy -c:a aac -ar 48000 \
  'audio_mix/{project_id}_music_mixed.mp4'
```

Music fade in: first 2s. Music fade out: last 3s.

Write `audio_mix/{project_id}_music_selected.md`:
```markdown
# Music Selection — {project_id}
**Track:** [filename]
**BPM:** [N]
**Mood:** [description]
**Why:** [one sentence]
**License:** [type]
```

## Step 5 — Assemble Final Video

Concatenate intro → content → outro (if applicable). Source for content is `music_mixed.mp4` if music was added, otherwise `overlaid.mp4`.

```bash
# Create concat list
echo "file 'intro/{project_id}_intro.mp4'" > /tmp/concat.txt
echo "file '{source_video}'" >> /tmp/concat.txt
echo "file 'outro/{project_id}_outro.mp4'" >> /tmp/concat.txt

ffmpeg -f concat -safe 0 -i /tmp/concat.txt \
  -c:v libx264 -preset slow -crf 18 \
  -c:a aac -ar 48000 \
  "{project_id}_final.mp4"
```

If no intro/outro: `final.mp4` = copy of content source.

---

## Step 6 — QC Verification (always runs)

Run all checks. Record pass/fail for each.

**1. Spec check:**
```bash
ffprobe -v quiet -print_format json -show_streams "{project_id}_final.mp4"
```
Pass: 1920×1080, H.264, AAC 48kHz, 30fps.

**2. Loudness:**
```bash
ffmpeg -i "{project_id}_final.mp4" -af ebur128=peak=true -f null - 2>&1 | grep "I:"
```
Pass: integrated LUFS between -15.5 and -12.5.

**3. Duration:**
Expected = overlaid duration + intro duration + outro duration. Pass: within ±1s.

**4. Intro/outro frames:**
```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/timeline_view.py" \
  "{project_id}_final.mp4" 0 3      # check intro
python3 "... timeline_view.py" "{project_id}_final.mp4" {end-3} {end}  # check outro
```

**5. Overlay spot-check:** Pick 3 random timestamps from overlay_check.md, sample each with timeline_view.py. Verify overlays are visible and correctly positioned.

**6. PTS check:**
```bash
ffprobe -show_packets -select_streams v:0 "{project_id}_final.mp4" 2>/dev/null | grep pts | head -20
```
Pass: no discontinuities (no large PTS jumps).
