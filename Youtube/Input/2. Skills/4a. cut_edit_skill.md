# Cut Edit Skill — How to Cut a Raw Video

Claude executes this sub-stage by default.

One raw video in, one clean cut video out. Follow these steps in order.

---

## Step 1 — Locate and inspect source

Find the normalized raw `.mp4` at `Youtube/Output/4. Editing/{project_id}/{project_id}_raw.mp4`.

```bash
ffprobe -v quiet -print_format json -show_streams "path/to/video.mp4"
```

Note: duration, resolution, fps, audio codec. Set `<project_id>` from the project folder.

Set `<edit_dir>` = `Youtube/Output/4. Editing/<project_id>/`

---

## Step 2 — Transcribe with ElevenLabs Scribe

Run the ElevenLabs Scribe helper (cached — skips if transcript already exists). API key is read from `~/.claude/.env`.

```bash
export ELEVENLABS_API_KEY=$(grep '^ELEVENLABS_API_KEY=' ~/.claude/.env | cut -d'=' -f2-)
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/transcribe.py" \
  "path/to/video.mp4" \
  --edit-dir "<edit_dir>" \
  --language en
```

Output: `<edit_dir>/<project_id>_transcript.json` — word-level timestamps. If the helper writes to a cache subfolder first, move or copy the normalized transcript to this canonical path.

---

## Step 3 — Pack transcript into reading view

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/pack_transcripts.py" \
  --edit-dir "<edit_dir>"
```

Output: `<edit_dir>/<project_id>_takes_packed.md` — phrase-level view with `[start-end]` timestamps.

---

## Step 4 — Pre-scan for problems

Read `<project_id>_takes_packed.md`. Note:
- Verbal slips, mis-speaks, double-takes
- Filler-heavy passages (um, uh, ah, like, you know, so, basically)
- Long silence gaps (>0.5s between phrases)

List findings in plain English. This feeds the cut strategy.

---

## Step 5 — Propose cut strategy

Describe in plain English (4–6 sentences):
- What gets removed (fillers, silences, bad takes)
- Any takes or beats being prioritised
- Estimated output duration
- Grade direction (if any)

**Wait for Daniel to confirm before touching the cut.**

---

## Step 6 — Drill into ambiguous moments

For any uncertain cut point, use the timeline viewer:

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/timeline_view.py" \
  "path/to/video.mp4" <start_seconds> <end_seconds>
```

This generates a filmstrip + waveform PNG for visual verification. Use it at decision points only — not as a scan tool.

---

## Step 7 — Build the EDL

Write `<edit_dir>/<project_id>_edl.json`. Each range must:
- Start and end on a word boundary from the transcript
- Include 50–80ms padding before the first word and after the last word
- Prefer silence gaps ≥ 400ms as cut targets
- Include `beat`, `quote`, and `reason` fields

```json
{
  "version": 1,
  "sources": {"<slug>": "/abs/path/to/video.mp4"},
  "ranges": [
    {
      "source": "<slug>",
      "start": 2.42,
      "end": 31.80,
      "beat": "INTRO",
      "quote": "first few words...",
      "reason": "Clean take, no fillers"
    }
  ],
  "grade": "none",
  "overlays": [],
  "subtitles": null,
  "total_duration_s": 0
}
```

Calculate and fill `total_duration_s` = sum of all `(end - start)` values.

---

## Step 8 — Render the cut

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/render.py" \
  "<edit_dir>/<project_id>_edl.json" \
  -o "Youtube/Output/4. Editing/<project_id>/<project_id>_cut.mp4"
```

The render helper: extracts per-segment clips with 30ms audio fades → lossless concat → applies grade → composite overlays → subtitles last.

---

## Step 9 — Self-evaluate

Run `timeline_view.py` on the **rendered output** (not the source) at every cut boundary (±1.5s window). Check:
- No visual flash or jump at the cut
- No waveform spike (audio pop)
- Grade consistency across segments

Also sample first 2s, last 2s, and 2–3 mid-points.

If anything fails: fix the EDL → re-render → re-check. Cap at 3 passes; flag remaining issues to Daniel.

```bash
ffprobe -v quiet -print_format json -show_streams \
  "Youtube/Output/4. Editing/<project_id>/<project_id>_cut.mp4"
```

Verify duration matches `total_duration_s` from the EDL (±1s tolerance).

---

## Step 10 — Persist session memory

Append to `<edit_dir>/<project_id>_project.md`:

```markdown
## Session N — YYYY-MM-DD

**Strategy:** one paragraph
**Decisions:** take choices, cuts, grade + why
**Reasoning log:** one-line rationale for non-obvious decisions
**Outstanding:** deferred items
```

Confirm the saved location of `<project_id>_cut.mp4` to Daniel.
