# Analytics Skill

Claude executes this stage by default.

Use this skill after a video has been published (Stage 7) to track its performance. Run `/analytics` — it handles everything automatically.

---

## 1. What Gets Tracked

### Per-video (at Day 7 and Day 30)

| Metric | What it tells you |
|--------|------------------|
| Views | Raw reach |
| Impressions | How often the thumbnail appeared in feeds |
| CTR | Thumbnail + title effectiveness. Below 4% = weak packaging |
| Avg view duration % | Algorithm's #1 signal. Below 40% = content or pacing issue |
| Watch time (min) | Absolute signal fed to the algorithm |
| Likes | Audience approval |
| Comments | Community engagement |
| Subscribers gained | How well the video converts viewers into followers |
| Top traffic source | Where viewers came from |

### Monthly channel rollup (last week of each month)
- Total views and watch time across all videos
- Net subscribers for the month
- Best video by avg view duration %, CTR, and subscribers gained

### Not tracked yet
- Revenue — add when channel is monetised
- Geographic breakdown — too granular for this stage
- Skool/client conversions — manual for now

---

## 2. Snapshot Schedule

YouTube Analytics data has a **2-3 day delay**. Snapshots are pulled:
- **Day 7** — early signal. Use it to decide if a thumbnail needs A/B testing
- **Day 30** — mature performance. The number that goes into the monthly rollup

The `/analytics` command calculates due dates automatically from the upload log.

## 5. Step-by-Step Workflow

**Step 1 — Enable YouTube Analytics API (once)**
Same Google Cloud project as the YouTube Data API. Go to **APIs & Services → Library → YouTube Analytics API → Enable**. The token will re-auth in the browser on the first run after this.

**Step 2 — Run `/analytics`**
The command:
1. Reads `upload_log.md` for all published project IDs, video IDs, and dates
2. Determines which Day 7 / Day 30 snapshots are due and missing
3. Fetches them via `fetch_analytics.py`
4. Generates the monthly rollup if it's the last week of the month

**Step 3 — Read the snapshot**
Open the generated `.md` file. Key questions:
- CTR below 4%? → Consider swapping the thumbnail
- Avg view duration below 40%? → Hook or pacing issue; review the script
- Top traffic source is YouTube Search? → Title/tags are working
- Top traffic source is Suggested? → Algorithm is picking it up

**Step 4 — Feed back into Ideation**
At the start of each month, use the monthly rollup to answer:
- What type of video performed best?
- What topic got the most subscribers?
- Use these signals to weight the next round of ideation (Stage 1)

## 6. Error Reference

| Error | Cause | Fix |
|-------|-------|-----|
| `No analytics data returned` | YouTube's 2-3 day delay | Wait and retry — the command skips and logs it |
| `YouTube Analytics API not enabled` | API not activated in Google Cloud | Enable it in APIs & Services → Library |
| `Token insufficient scopes` | Old token from before analytics scope was added | Delete `token.json` and re-run — browser re-auth triggers automatically |
| `HTTP 403` | Quota exceeded or wrong account | Check Google Cloud console for quota usage |
