# Overlay Skill (Stage 5a) — How to Add Overlays to a Cut Video

Codex executes this stage.

One cut video in, one overlaid video out. Follow these steps in order.

---

## Step 1 — Locate and inspect the cut video

Find the cut `.mp4` at `Youtube/Output/4. Editing/{project_id}/{project_id}_cut.mp4`.

```bash
ffprobe -v quiet -print_format json -show_streams "Youtube/Output/4. Editing/{project_id}/{project_id}_cut.mp4"
```

Note: duration, resolution (must be 1920×1080), fps (30). Set `<project_id>` from the project folder.

`<edit_dir>` = `Youtube/Output/4. Editing/{project_id}/`  
The EDL and session memory live there.

`<visuals_dir>` = `Youtube/Output/5. Visuals/{project_id}/`

---

## Step 2 — Receive the overlay brief

Ask Daniel (or read from the session brief) what overlays are needed. For each overlay capture:
- **Content**: what text, data, or graphic
- **Timing**: when it appears in the OUTPUT timeline (seconds)
- **Duration**: how long it should be visible
- **Style**: brand colors, font, palette (if not already in `project.md`)

If style hasn't been defined yet, propose a palette and wait for confirmation before building.

---

## Step 3 — Choose the rendering tool

| Tool | Use when |
|------|----------|
| **HyperFrames** | Data cards, stat callouts, step lists, kinetic text, agent-written HTML from scratch |
| **Remotion** | Reusing existing compositions (`AiBusinessChip`, `GoodBadUglyCards`, `WorkflowMap`) or building new React/TSX components |

Check `Youtube/Input/5. Tools/5. visuals_tools/remotion-visuals/src/Composition.tsx` for existing Remotion compositions before building new ones.

---

## Step 4 — Build each animation slot

Create a slot directory per overlay:
```
<visuals_dir>/animations/slot_<id>/
```

### If using HyperFrames:

```bash
npx --yes hyperframes init "<visuals_dir>/animations/slot_<id>" \
  --example blank --non-interactive --skip-skills
```

Write the HTML composition into the slot directory. Then validate:

```bash
cd "<visuals_dir>/animations/slot_<id>"
npx hyperframes lint
npx hyperframes validate
```

Both must pass. Then render:

```bash
npx --yes hyperframes render "<visuals_dir>/animations/slot_<id>" \
  -o "<visuals_dir>/animations/slot_<id>/{project_id}_overlay_slot_<id>.mp4"
```

Use `--format webm` instead of `.mp4` when alpha channel is needed.

### If using Remotion:

For new compositions, scaffold inside the slot dir:
```bash
cd "<visuals_dir>/animations/slot_<id>"
npx create-video@latest
```

For existing `remotion-visuals/` compositions, render from the project:
```bash
cd "Youtube/Input/5. Tools/5. visuals_tools/remotion-visuals"
npx remotion render <CompositionId> "<visuals_dir>/animations/slot_<id>/{project_id}_overlay_slot_<id>.mp4"
```

Always verify the rendered overlay with `ffprobe`:
```bash
ffprobe -v quiet -print_format json -show_streams \
  "<visuals_dir>/animations/slot_<id>/{project_id}_overlay_slot_<id>.mp4"
```

Check: duration matches intended overlay window, resolution is 1920×1080, fps is 30.

---

## Step 5 — Multiple overlays: spawn parallel sub-agents

If there are 2+ overlays, spawn one sub-agent per slot via the `Agent` tool. Each sub-agent brief must be fully self-contained (include: goal, output path, spec, style palette, frame-by-frame timeline, anti-list). Run all slots in parallel — wall time ≈ slowest slot.

---

## Step 6 — Update the EDL with overlay entries

Update `<edit_dir>/<project_id>_edl.json` to add overlay references:

```json
"overlays": [
  {
    "file": "<visuals_dir>/animations/slot_001/{project_id}_overlay_slot_001.mp4",
    "start_in_output": 4.5,
    "duration": 6.0
  }
]
```

`start_in_output` is the position in the **output timeline** (after cuts), not the source timeline.

---

## Step 7 — Render the final composite

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/render.py" \
  "<edit_dir>/<project_id>_edl.json" \
  -o "Youtube/Output/5. Visuals/{project_id}/{project_id}_overlaid.mp4"
```

The render helper composites overlays using `setpts=PTS-STARTPTS+T/TB` (correct PTS offset). Subtitles are applied last if included in the EDL.

---

## Step 8 — Self-evaluate

Sample the output at:
- Each overlay start and end point (check alignment, no drift)
- First 2s and last 2s of the video
- 2–3 mid-points (grade consistency, subtitle readability if applicable)

```bash
ffprobe -v quiet -print_format json -show_streams \
  "Youtube/Output/5. Visuals/{project_id}/{project_id}_overlaid.mp4"
```

Verify: duration matches expected, resolution 1920×1080, H.264 + AAC 48kHz.

---

## Step 9 — Persist session memory

Append to `<visuals_dir>/<project_id>_overlay_check.md`:

```markdown
## Overlay Session N — YYYY-MM-DD

**Overlays:** list each slot, tool used, timing, content
**Style:** palette, font, easing choices
**Reasoning:** why each overlay was placed where it was
**Outstanding:** deferred overlays or style changes
```

Confirm the saved location of `{project_id}_overlaid.mp4` to Daniel.
