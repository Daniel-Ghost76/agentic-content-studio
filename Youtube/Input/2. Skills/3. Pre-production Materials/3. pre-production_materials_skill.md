# Pre-production Materials Skill

This skill defines the Stage 3 workflow. Codex uses it to create the slides and visual materials Daniel records with.

## What This Stage Does

Stage 3 turns the Stage 2 Visuals Brief into ready-to-use pre-recording assets.

Create:

- slides for screen sharing
- simple diagrams and visual explainers
- static AI-generated visual references when needed
- production notes for timing, dimensions, and recording use

Do not create:

- cut edits
- subtitles
- overlays composited onto recorded footage
- final rendered videos
- thumbnails for Review

Those belong to later stages.

## Workflow

1. Read the Visuals Brief from `Youtube/Input/4. Resources/3. pre-production_materials_resources/`.
2. Read the Visual Production Library overview, selection matrix, and pre-production sub-library:
   - `Youtube/Input/4. Resources/0. visual_production_library/README.md`
   - `Youtube/Input/4. Resources/0. visual_production_library/selection_matrix.md`
   - `Youtube/Input/4. Resources/0. visual_production_library/01_pre_production_materials/`
3. Read the approved script from `Youtube/Output/2. Scripts/` if needed for context.
4. Read the `project_id` from the upstream Visuals Brief or Stage 2 script folder.
5. Create an output folder in `Youtube/Output/3. Pre-production Materials/{project_id}/`.
6. Choose an approved visual capsule from `01_pre_production_materials/`. If no capsule fits, ask Daniel whether to build a new draft capsule before generating the asset.
7. Build the requested slides or visual assets.
8. Use deterministic rendering for text-heavy slides whenever possible so text, spacing, and dimensions are reliable.
9. Use AI image generation only when the visual needs an illustrative bitmap, style reference, or generated scene.
10. Save all final assets and a `{project_id}_production_note.md` in the episode output folder.
11. Verify dimensions, legibility, and exact text before handing back to Daniel.

## Style Priority

The Visuals Brief wins. If it says minimal dark slides, use minimal dark slides. If it asks for the approved teal HUD language, use the approved visual references from Stage 5 resources.

Default pre-production style:

- clean
- readable
- high contrast enough for recording
- simple composition
- no decorative complexity unless it helps Daniel explain the idea

When a library capsule is used, the capsule wins over this default unless Daniel's latest instruction says otherwise.

## File Naming

Use the canonical project prefix:

```text
{project_id}_slide_01.png
{project_id}_slide_02.png
{project_id}_diagram_01.png
{project_id}_production_note.md
```

For generated image concepts, keep a matching prompt file:

```text
{project_id}_diagram_01.png
{project_id}_prompt_01.txt
```

## Quality Bar

- Every asset has a clear recording purpose.
- The chosen Visual Production Library capsule is named in the production note.
- Viewer-facing text is large and readable.
- Exact on-screen text from the brief is not paraphrased.
- The output folder is self-explanatory.
- Nothing depends on an undocumented tool or hidden manual step.
