# Overlay Identifier Skill — How to Map Visual Events

Claude executes this stage. One transcript read, one overlay map out. Follow these steps in order.

---

## Step 1 — Run Setup Questions

Ask Daniel the five setup questions in `4c. overlay_identifier_sub-agent.md`. Record all answers before opening any file.

---

## Step 2 — Read the Cut Video and Transcript

Inspect the cut:
```bash
ffprobe -v quiet -print_format json -show_streams \
  "Youtube/Output/4. Editing/{project_id}/{project_id}_cut.mp4"
```

Note total duration. This determines how many zooms are allowed (1 per 60s max) and what "hook" and "outro" mean time-wise.

Read `{project_id}_takes_packed.md` fully. This is your primary source for all five overlay types.

Also read `{project_id}_transcript.json` to get word-level timestamps for precise caption and zoom placement.

Read the Visual Production Library selection matrix before mapping visual events:

```text
Youtube/Input/4. Resources/0. visual_production_library/selection_matrix.md
```

---

## Step 3 — Map Caption Overlays (Type 1)

Target: 10–15% of total spoken words. Apply the agreed density level from setup.

Selection criteria (in priority order):
1. The single most important sentence in the hook (first 30s)
2. The single most important statement in the outro (last 30s)
3. Numbers, statistics, or specific claims ("90% of time is wasted")
4. Contrasts or pivots ("But here's the problem...")
5. Strong imperatives or calls to action

For each selection, write a stub. Use the exact spoken words — do not paraphrase. Timestamps from transcript.

Example output block:
```
### Caption Overlays

[00:04.2] CAPTION | text: "90% of what a web agent does is completely wasted" | duration: 4.0s | position: lower-third | library: 05_emphasis_word_overlays | capsule: draft-or-approved-capsule-name
[00:31.5] CAPTION | text: "We fixed this" | duration: 2.5s | position: center | library: 05_emphasis_word_overlays | capsule: draft-or-approved-capsule-name
[04:22.1] CAPTION | text: "One pipeline. Zero manual steps." | duration: 3.5s | position: lower-third | library: 05_emphasis_word_overlays | capsule: draft-or-approved-capsule-name
```

## Step 4 — Map Zoom Keyframes (Type 2)

Maximum 1 zoom per 60 seconds of output duration. Identify the strongest moments — typically:
- Landing a key claim (payoff word)
- Transitioning from problem to solution
- The single strongest emotional beat

For each, find the exact timestamp of the payoff word from the transcript, then start the zoom ease-in 1 second before.

```
### Zoom Keyframes

[00:30.8] ZOOM | peak: 1.10x | ease_in: 0.8s | hold: 1.2s | ease_out: 0.7s | reason: "landing the hook payoff — 'We fixed this'"
[02:15.3] ZOOM | peak: 1.08x | ease_in: 0.6s | hold: 1.0s | ease_out: 0.5s | reason: "pivot to demo reveal"
```

If you identify more zoom candidates than the budget allows, list them all and note which ones are over budget. Ask Daniel to choose.

---

## Step 5 — Map HyperFrames Overlays (Type 3)

Look for moments where Daniel:
- Lists numbered steps
- States a statistic or metric
- Explains a concept that has a diagram or visual equivalent
- Refers to a before/after comparison

Each stub needs: what it shows, not how to build it. Agent 5 writes the HTML.

```
### HyperFrames Overlays

[01:23.0] HYPERFRAMES | type: step-list | content: "Step 1: Ingest -> Step 2: Transcribe -> Step 3: Cut" | duration: 8s | position: right-panel | library: 02_partial_overlays_hyperframes | capsule: draft-or-approved-capsule-name
[03:05.2] HYPERFRAMES | type: stat-card | content: "Before: 4 hours / After: 22 minutes" | duration: 5s | position: lower-right | library: 02_partial_overlays_hyperframes | capsule: draft-or-approved-capsule-name
```

## Step 6 — Map Higgsfield Clips (Type 4)

Limit: the cap confirmed in setup questions. Review entire video first, list all candidates, then select the best ones up to the cap.

Each entry needs a concept and prompt direction — not a finished Higgsfield prompt. Agent 5 finalizes the prompt.

```
### Higgsfield Full-Screen Clips

[02:44.0] HIGGSFIELD | duration: 4s | concept: "AI agents autonomously working through a codebase at night" | library: 04_higgsfield_visuals | capsule: draft-or-approved-capsule-name | prompt-direction: "glowing terminal windows, abstract green/teal code flows, no humans visible, premium cinematic look"
[04:10.5] HIGGSFIELD | duration: 3s | concept: "A clean automated pipeline running perfectly" | library: 04_higgsfield_visuals | capsule: draft-or-approved-capsule-name | prompt-direction: "smooth flowing data visualization, dark background, teal accents, futuristic but minimal"
```

If candidates exceed the cap: list all with a priority ranking (most impactful first) and let Daniel choose.

---

## Step 7 — Map Additional Materials (Type 5)

Check `originals/` for any files that aren't the main face cam or screen recording. If none exist, skip this section entirely.

If extras exist, identify where each one fits in the timeline based on what Daniel is describing at that moment.

```
### Additional Materials

[03:10.0] MATERIAL | file: "originals/dashboard_demo.mp4" | duration: 8s | usage: full-replace (cut away from face cam entirely during this section)
```
