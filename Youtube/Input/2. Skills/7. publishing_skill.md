# Publishing Skill

Claude executes this stage by default. Codex may orchestrate mobile publishing when Daniel asks from the Codex app.

Use this skill when a video has been approved in the Review stage and needs to be scheduled on YouTube. The canonical publish pair must be in `Youtube/Output/7. Publishing/{project_id}/` before this skill begins.

---

## 1. What "Ready to Publish" Means

A video is ready when both conditions are met:

1. The final video file is `Youtube/Output/7. Publishing/{project_id}/{project_id}_publish.mp4`
2. The metadata file is `Youtube/Output/7. Publishing/{project_id}/{project_id}_metadata.yaml`

Videos in `Youtube/Output/7. Publishing/{project_id}/published/` have already been uploaded — do not re-process them.

---

## 2. Folder and File Naming

```
Youtube/Output/7. Publishing/
  {project_id}/
    {project_id}_publish.mp4
    {project_id}_metadata.yaml
    published/
      {project_id}_publish.mp4
      {project_id}_metadata.yaml
```

The project ID must match the upstream Review package. Current Episode 1 is `01-transcript-youtube`.

---

## 3. YAML Metadata Format

Required fields are `title`, `description`, `category_id`, and `made_for_kids`.

```yaml
title: "Your video title (max 100 characters)"

description: |
  Opening hook — first 150 characters appear in search results and email notifications.

  Timestamps
  0:00 Intro
  [add remaining timestamps]

  Links

  #Hashtag1 #Hashtag2 #Hashtag3

category_id: "28"
made_for_kids: false
```

Do **not** include `tags` or `privacy` fields — all uploads are scheduled automatically.

---

## 4. Scheduling Rules

All videos are scheduled, never published immediately.

| Videos ranked | Slot |
|---------------|------|
| 1st, 2nd, 3rd | Next 3 Fridays at 4:00 PM London |
| 4th and beyond | Consecutive Tuesdays at 4:00 PM London |

Timezone: **Europe/London** (automatically handles GMT/BST).

Higher-ranked videos always publish before lower-ranked ones — a Tuesday video can never precede a Friday video with a higher rank.

---

## 5. Relevance Ranking

Before assigning schedule slots, rank videos by current relevance:

**Time-sensitive (rank first):**
- Tied to recent AI model releases, tool launches, platform updates
- Responds to current events or trending topics
- News will be stale within 1–2 weeks

**Evergreen (keep original recording order):**
- General tutorials, skill walkthroughs, process explainers
- Content that is as relevant in 3 months as it is today

When uncertain, default to original recording order (by filename date).

## 6. Step-by-Step Upload Workflow

**Step 1 — Verify credentials**
Check that `Youtube/Input/4. Resources/7. publishing_resources/credentials.json` exists. If not, follow the one-time Google Cloud setup (Section 8).

**Step 2 — Prepare companion YAML files**
For each video without a `.yaml`, create one using the template in Section 3. Fill in title and description — pull copy from the approved script in `Youtube/Output/2. Scripts/` if available.

**Step 3 — Run `/publish`**
In Claude Code, run the `/publish` slash command. The full workflow runs automatically:
- Scans for eligible pairs
- Ranks by relevance
- Calculates schedule slots
- Shows confirmation table
- Uploads and schedules on YouTube
- Moves files to `published/`
- Logs to Analytics

**Step 4 — Verify in YouTube Studio**
After upload, open YouTube Studio and confirm:
- Title and description are correct
- Thumbnail set (set manually if needed)
- Schedule date/time is correct
- No Content ID claims

**Step 5 — Pass URL to Distribution**
Hand the YouTube URL to Stage 8 (Distribution) for cross-platform sharing.

## 7. Error Reference

| Error | Cause | Fix |
|-------|-------|-----|
| `credentials.json not found` | File missing | Complete Google Cloud setup (Section 8) |
| `token.json` invalid | OAuth token expired or revoked | Delete `token.json`, run `/publish` again for browser re-auth |
| `HTTP 403 quotaExceeded` | Daily API quota hit | Wait 24 hours; daily quota is 10,000 units |
| `HTTP 400 invalidTitle` | Title > 100 chars or forbidden characters | Shorten/fix in the YAML |
| `HTTP 400 invalidPublishAt` | Schedule time is in the past or < 5 min from now | Use a future time |
| Upload stalls | Network interruption | Script retries up to 10× with exponential backoff |
| Thumbnail rejected | File > 2 MB or wrong format | Compress JPEG or use PNG under 2 MB |

## 8. One-Time Google Cloud Setup

Complete once before the first upload:

1. Go to **console.cloud.google.com** → create a new project (name it anything, e.g. "YouTube Publisher")
2. Go to **APIs & Services → Library** → search **YouTube Data API v3** → **Enable**
3. Go to **APIs & Services → OAuth consent screen**:
   - User type: **External**
   - Fill in app name and your email
   - Add scope: `youtube.upload`
   - Add yourself as a **Test user** (required while the app is unverified)
4. Go to **Credentials → Create Credentials → OAuth 2.0 Client ID**:
   - Application type: **Desktop app**
   - Click **Create** → **Download JSON**
5. Rename the downloaded file to `credentials.json`
6. Place it at: `Youtube/Input/4. Resources/7. publishing_resources/credentials.json`
7. Run `/publish` — a browser window opens for Google sign-in
8. After signing in, `token.json` is saved in the same folder — future runs are automatic

**Keep both files private.** Never commit or share `credentials.json` or `token.json`.
