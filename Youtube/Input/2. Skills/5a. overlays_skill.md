## Step 3 — Screen Recording Cleanup (if present)

If the manifest has a screen recording source:

```bash
# Scale to 1920×1080, boost contrast for UI readability, crop menu bar if visible
ffmpeg -i "normalized/{project_id}_screen_rec.mp4" \
  -vf "crop=iw:ih-25:0:25,scale=1920:1080:force_original_aspect_ratio=decrease,pad=1920:1080:(ow-iw)/2:(oh-ih)/2,eq=contrast=1.08:saturation=0.92,format=yuv420p" \
  -c:v libx264 -preset slow -crf 18 \
  -c:a aac -ar 48000 \
  "screen_rec/{project_id}_screen_rec_clean.mp4"
```

Verify with `ffprobe`. Sample a frame to confirm menu bar is cropped and text is sharp. Adjust crop values if the menu bar is taller or absent.

## Step 5 — Build All Overlay Slots in Parallel

Spawn one sub-agent per slot using the Agent tool. All slots run simultaneously — wall time ≈ slowest slot. Each sub-agent receives a self-contained brief.

### Caption Overlays and HyperFrames Motion Graphics (HyperFrames)

For each CAPTION or HYPERFRAMES entry in the map, the sub-agent builds a HyperFrames composition:

```bash
# Initialize slot
npx --yes hyperframes init "animations/slot_001" \
  --example blank --non-interactive --skip-skills

# Write HTML composition into slot_001/ directory
# (sub-agent writes the full HTML/CSS/GSAP for this specific stub)

# Validate
npx hyperframes lint
npx hyperframes validate

# Render
npx --yes hyperframes render . -o {project_id}_overlay_slot_001.mp4
# Use --format webm for alpha channel if needed
```

**Caption overlay style (glass panel):**
- Background: `rgba(10, 20, 30, 0.65)` with `backdrop-filter: blur(12px)`
- Border: `1px solid rgba(100, 200, 210, 0.25)`
- Text: white, bold, 48–64px depending on length
- Animation: slide up + fade in over 0.4s (`ease_out_cubic`), hold, fade out over 0.3s
- Hold final frame for ≥0.5s before fade

**HyperFrames motion graphic style (HUD teal glass):**
- Same glass panel base + teal glow accent (`rgba(0, 180, 200, 0.6)`)
- Use dotted grid background texture
- Animation: soft scale-in from 0.95 over 0.5s

### Higgsfield Full-Screen Clips

For each HIGGSFIELD entry, the sub-agent:
1. Reads the stub's concept and prompt-direction
2. Writes a finalized Higgsfield prompt and saves as `higgsfield/slot_00N_prompt.txt`
3. If the stub specifies a starting image: generate it first (DALL-E via OpenAI API or use an approved visual as reference)
4. Calls Higgsfield to generate the clip
5. Saves output to `higgsfield/slot_00N_clip.mp4`
6. Flags output to Codex/Daniel for review before compositing

```
Use: mcp__claude_ai_Higgsfield__generate_video
```

Review generated clip with `timeline_view.py` before compositing. Flag if generation looks wrong.

## Step 7 — Update EDL with Overlay Entries

Add all overlay slots to `{project_id}_edl.json`:

```json
"overlays": [
  {
    "file": "Youtube/Output/5. Visuals/{project_id}/animations/slot_001/{project_id}_overlay_slot_001.mp4",
    "start_in_output": 4.2,
    "duration": 4.0
  },
  {
    "file": "Youtube/Output/5. Visuals/{project_id}/higgsfield/slot_001_clip.mp4",
    "start_in_output": 164.0,
    "duration": 4.0,
    "full_screen": true
  }
]
```

`start_in_output` is position in OUTPUT timeline (post-cuts), not source timeline.

## Step 8 — Composite Final Video

```bash
python3 "Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/render.py" \
  "Youtube/Output/4. Editing/{project_id}/{project_id}_edl.json" \
  -o "Youtube/Output/5. Visuals/{project_id}/{project_id}_overlaid.mp4"
```

Subtitles (if in EDL) applied LAST.
