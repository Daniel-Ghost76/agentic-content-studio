Walk the entire workspace tree. For each folder and file, check its name against the rules below. Collect all violations before making any changes.

### File Naming Rules

**`Youtube/Input/1. Sub-agents/`:**
Pattern: `{N}. {stage}_sub-agent.md` or `{N}a. {stage}_sub-agent.md`
Exceptions: `cleanup_sub-agent.md`, `.audit_sub-agent.md` — skip both

**`Youtube/Input/2. Skills/`:**
Pattern: `{N}. {stage}_skill.md` or `{N}a. {stage}_skill.md`
Exception: `2. scripting_skill..md` — skip

**`Youtube/Input/3. Rules/`:**
Pattern: `{N}. {stage}_rules.md`

**Files inside `Youtube/Output/{stage}/{project_id}/`:**
Pattern: `{project_id}_{purpose_suffix}.{ext}`
Exceptions: `upload_log.md` at the Stage 9 root level, anything inside `originals/` or `published/`

### Auto-Fix Rules

Auto-rename only when ALL of the following are true:
1. The violation is unambiguous (only one possible correct name)
2. The correct name does not already exist at that path (would not cause a collision)
3. The file is not in the Hard Exceptions list

For every auto-rename:
1. Log: `[rename] {old_path} → {new_path}`
2. Execute: `mv "{old_path}" "{new_path}"`
3. Confirm: `[done]` or log any error

If any of the three conditions above are not met, add the item to the **Needs Review** list and do not touch it.

## Module C — MD Content Audit

Scan every `.md` file inside `Youtube/Input/1. Sub-agents/`, `Youtube/Input/2. Skills/`, and `Youtube/Input/3. Rules/` for broken file path references.

**Steps:**

1. For each `.md` file, read its full contents.

2. Extract every line that contains a file path reference. Indicators:
   - Starts with `Youtube/` or contains `` `Youtube/ ``
   - Contains `~/.claude/` or `/Users/danieldanut/`
   - Backtick-wrapped paths like `` `Youtube/Output/...` ``

3. For each extracted path:
   - Strip any template variables: remove `{project_id}`, `{stage}`, `{N}`, etc. — what remains is the parent path to check
   - If the stripped path still contains `{`, it's a template — skip it (it's correct by design)
   - If the stripped path is fully concrete, check whether it exists on disk with `test -e` or `ls`

4. Flag any concrete path that does not exist. Record:
   - Which `.md` file contains the reference
   - The line number
   - The broken path as written
   - The suggested correct path (if determinable by searching nearby directories)

**Do not auto-edit any `.md` file.** Report only.

- Broken path: `Youtube/Quick Access/youtube_ideas_30_order.pdf`

## Module D — Output Artifact Audit

Check every file inside `Youtube/Output/{stage}/{project_id}/` subdirectories.

**Rules:**
- Every file at the project folder level must start with `{project_id}_`
- The `{project_id}` in the filename must match the containing folder name exactly
- The `project_id` used in any filename must also exist as a folder in `Youtube/Output/1. Ideation/`

**Exceptions — skip these:**
- `upload_log.md` at the `Youtube/Output/9. Analytics/` root (not inside a project folder)
- Any file inside an `originals/` subdirectory
- Any file inside a `published/` subdirectory
- `.DS_Store` and other macOS metadata files

**Steps:**

1. Walk every file under `Youtube/Output/`
2. For each file that is not in an exception path:
   - Check that the filename starts with the parent `project_id` folder name + `_`
   - Check that the `project_id` prefix exists in Stage 1 (Ideation)
3. Flag violations — do not auto-rename artifacts (content matters, names may be referenced elsewhere)

```

──────────────────────────────
A. Naming:      {N} renamed, {N} need review

C. MD refs:     {N} broken references
```

Send with:
```bash
curl -s -X POST "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendMessage" \
  -d chat_id="${TELEGRAM_CHAT_ID}" \
  -d text="..." \
  -d parse_mode="HTML"
```

```
⚠ Telegram skipped — TELEGRAM_CHAT_ID not found in ~/.claude/.env. Add it to enable notifications.
```

If `TELEGRAM_BOT_TOKEN` is also missing, log the same note.
