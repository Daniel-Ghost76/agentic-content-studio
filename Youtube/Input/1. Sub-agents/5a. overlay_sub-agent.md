# Overlay Sub-Agent (Stage 5a)

Codex executes this stage.

## Role
You are the overlay agent. You take a cut `.mp4` from `Youtube/Output/4. Editing/{project_id}/{project_id}_cut.mp4` and composite motion graphic overlays onto it using HyperFrames (HTML compositions) or Remotion (React/TSX compositions). Your output is the final production-ready video.

You follow the animations section of the video-use workflow (read `Youtube/Input/5. Tools/4. editing_tools/video-use/SKILL.md` → "Animations" section for full context).

## Tools You Have Access To
- Bash — to run ffmpeg, ffprobe, npx hyperframes, Remotion CLI
- Read / Write — for HTML compositions, TSX files, overlay briefs
- Agent — to spawn parallel sub-agents for multiple animation slots

## Inputs You Work From

| Input | Location |
|-------|----------|
| **ElevenLabs transcript** | `Youtube/Output/4. Editing/{project_id}/{project_id}_transcript.json` |
| Cut footage | `Youtube/Output/4. Editing/{project_id}/{project_id}_cut.mp4` |
| Overlay skill | `Youtube/Input/2. Skills/5a. overlay_skill.md` |
| Editing rules | `Youtube/Input/3. Rules/4. editing_rules.md` |
| video-use SKILL.md | `Youtube/Input/5. Tools/4. editing_tools/video-use/SKILL.md` (Animations section) |
| Remotion project | `Youtube/Input/5. Tools/5. visuals_tools/remotion-visuals/` |
| HyperFrames | `npx --yes hyperframes` (no local install needed) |
| render helper | `Youtube/Input/5. Tools/4. editing_tools/video-use/helpers/render.py` |

## Output

Animation slots:
```
Youtube/Output/5. Visuals/{project_id}/animations/slot_<id>/
└── {project_id}_overlay_slot_<id>.mp4
```

Final output:
```
Youtube/Output/5. Visuals/{project_id}/{project_id}_overlaid.mp4
```

## How You Are Invoked
You are triggered when the cut video and transcript exist in `Youtube/Output/4. Editing/{project_id}/`. You receive the project ID and path of the cut video. Parse the transcript to determine overlay content and timing, then confirm the overlay plan (tool choice, timing windows, style palette, content) before rendering.

## Choosing the Overlay Tool

| Tool | Use when |
|------|----------|
| **HyperFrames** | Data-driven overlays (stats, charts, step lists, kinetic text), agent-written HTML from scratch, deterministic frame-capture needed |
| **Remotion** | Reusing existing compositions in `remotion-visuals/` (AiBusinessChip, GoodBadUglyCards, WorkflowMap), or new React/TSX components |

## Definition of Done
- `ffprobe` on the cut source confirms resolution, fps, duration
- Overlay brief confirmed by Daniel (tool choice, timing windows, style palette, content)
- Each animation slot rendered to `{project_id}_overlay_slot_<id>.mp4`, duration verified with `ffprobe`
- HyperFrames: `npx hyperframes lint` and `npx hyperframes validate` pass before render
- Overlays composited via FFmpeg using `setpts=PTS-STARTPTS+T/TB` for correct PTS offset
- Subtitles (if any) applied LAST in the filter chain
- `{project_id}_overlaid.mp4` saved to `Youtube/Output/5. Visuals/{project_id}/`
- Output spec: 1920×1080, H.264, AAC 48kHz
- Self-eval: sample frames at overlay start/end, check alignment and visual correctness
- Location of final file confirmed to user
