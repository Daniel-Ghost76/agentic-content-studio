# Publishing Sub-Agent

Claude executes this stage by default. OpenClaw (Astra) may orchestrate mobile publishing when Daniel triggers it via Telegram.

Scan the Publishing folder, rank videos by relevance, schedule them to the correct Friday/Tuesday slots, upload to YouTube, and log results.

Videos are **never published immediately** — every upload is scheduled.

---

## Role

The Publishing Sub-Agent is responsible for taking a finalized, reviewed video and getting it live on YouTube. It owns Stage 7 of the content pipeline — from the moment a video file lands in the Publishing folder to the moment a YouTube URL is confirmed and the Distribution stage is handed off.

---

## Trigger

Activate this sub-agent when:
- A canonical video pair has been moved to `Youtube/Output/7. Publishing/{project_id}/` after passing Review
- The user runs `/publish` in Claude Code
- Requested explicitly by the user

---

## Inputs

| Input | Source |
|-------|--------|
| Final video file | `Youtube/Output/7. Publishing/{project_id}/{project_id}_publish.mp4` |
| Metadata file | `Youtube/Output/7. Publishing/{project_id}/{project_id}_metadata.yaml` |

| Approved thumbnail (optional) | `Youtube/Output/6. Review  /{project_id}/{project_id}_thumbnail.png` |
| OAuth credentials | `Youtube/Input/4. Resources/7. publishing_resources/credentials.json` |

---

## Outputs

| Output | Destination |
|--------|-------------|
| Uploaded video (moved) | `Youtube/Output/7. Publishing/{project_id}/published/` |
| YouTube URL | Passed to Distribution stage |
| Upload log entry | `Youtube/Output/9. Analytics/upload_log.md` |

---

## Out of Scope

- Video editing or re-encoding (handled in Stage 4)
- Thumbnail creation (handled in Stage 6 — Review)
- Social media posting (handled in Stage 8 — Distribution)
- Analytics review (handled in Stage 9)

## Step 1 — Check credentials

Run:
```bash
ls "Youtube/Input/4. Resources/7. publishing_resources/credentials.json"
```

If the file does not exist, stop and print:
```
YouTube API credentials not found.

One-time setup:
  1. Go to console.cloud.google.com → create a new project
  2. Enable "YouTube Data API v3" (APIs & Services → Library)
  3. Credentials → Create Credentials → OAuth 2.0 Client ID → Desktop app → Create
  4. Download JSON → rename to credentials.json
  5. Place at: Youtube/Input/4. Resources/7. publishing_resources/credentials.json

Run /publish again — a browser window opens for Google sign-in on first use.
```

**Log:** `[credentials] credentials.json found`

## Step 2 — Check dependencies

The publishing scripts use a dedicated virtual environment at `Youtube/Input/4. Resources/7. publishing_resources/venv/`.
Always invoke scripts with that venv's Python:

```
VENV_PYTHON="Youtube/Input/4. Resources/7. publishing_resources/venv/bin/python3"
```

If the venv does not exist, create it and install dependencies:
```bash
python3 -m venv "Youtube/Input/4. Resources/7. publishing_resources/venv"
"Youtube/Input/4. Resources/7. publishing_resources/venv/bin/pip" install -r "Youtube/Input/4. Resources/7. publishing_resources/requirements.txt"
```

---

## Step 3 — Scan Publishing folder

Run this script to identify eligible pairs — do not manually scan the folder:
```bash
python3 "Youtube/Input/5. Tools/7. publishing_tools/validate_publish_pairs.py"
```
Present its output table to Daniel directly.

If the script exits with "No videos ready to publish", stop and tell Daniel.

**Before uploading any pair, validate its YAML:**
```bash
python3 "Youtube/Input/5. Tools/7. publishing_tools/validate_yaml.py" \
  "Youtube/Output/7. Publishing/{project_id}/{project_id}_metadata.yaml"
```
If validate_yaml.py returns FAIL, show Daniel the issues and do not proceed with upload until they are fixed.

---

## Step 4 — Relevance ranking

Read each eligible YAML file. For each video, briefly assess:
- Is it tied to recent news, product launches, AI model releases, platform changes, or current events? → **time-sensitive** (prioritise)
- Is it a general tutorial, evergreen process, or skill walkthrough? → **evergreen** (keep original order)

**Ranking rules:**
1. Time-sensitive videos rank first, ordered by how immediate/perishable the news is
2. Evergreen videos keep their original recording order (by filename date)
3. If uncertain, default to original recording order

**Log each assessment:**
```
[rank] "Day 1 of Building an AI Business from Scratch" — evergreen (build-in-public series) → rank 1 (only video)
```

If there is only one video, skip the ranking discussion and move on.

---

## Step 5 — Calculate schedule slots

Run:
```bash
"Youtube/Input/4. Resources/7. publishing_resources/venv/bin/python3" "Youtube/Input/4. Resources/7. publishing_resources/schedule_slots.py" <N>
```
where `<N>` is the number of eligible videos.

Capture the output — one ISO 8601 datetime per line. These are the schedule slots in ranked order:
- Slots 1–3 → next 3 Fridays at 4:00 PM London time
- Slots 4+ → consecutive Tuesdays at 4:00 PM London time

**Log:**
```
[schedule] Slot 1 → 2026-05-16T16:00:00+01:00  (Friday)
[schedule] Slot 2 → 2026-05-23T16:00:00+01:00  (Friday)
```

---

Print a clear table with ranked order, schedule assignment, and key metadata:

┌─────────────────────────────────────────────────────────────────────────────┐
│  PUBLISH QUEUE                                                              │
├────┬───────────────────────────────────────────────┬────────────────────────┤

Then proceed directly to Step 7.

## Step 7 — Upload each video

For each video in ranked order, run:

```bash
"Youtube/Input/4. Resources/7. publishing_resources/venv/bin/python3" "Youtube/Input/4. Resources/7. publishing_resources/upload_to_youtube.py" \
  --video "/full/path/to/video.mov" \
  --metadata "/full/path/to/video.yaml" \
  --schedule-at "2026-05-16T16:00:00+01:00"
```

Use the absolute path for `--video` and `--metadata`. Use the slot datetime from Step 5 for `--schedule-at`.

Parse the output for:
- `VIDEO_ID=` → capture the video ID
- `VIDEO_URL=` → capture the URL
- `SCHEDULED_AT=` → capture the scheduled time

**Log after each upload:**
```
[upload] "Day 1 of Building an AI Business from Scratch"
[upload]   Video ID:     abc123xyz
[upload]   URL:          https://www.youtube.com/watch?v=abc123xyz
[upload]   Scheduled:    2026-05-16T16:00:00+01:00 (Fri 16 May 4:00 PM London)
[upload]   Status:       SUCCESS
```

If the upload fails, log the error and continue to the next video. Do not move files for failed uploads.

---

## Step 8 — Move files to published/

After each successful upload, move the video file and its companion `.yaml` to the same project's published folder:
```
Youtube/Output/7. Publishing/{project_id}/published/
```

Create the `published/` subfolder if it does not exist.

```bash
mkdir -p "Youtube/Output/7. Publishing/{project_id}/published"
mv "Youtube/Output/7. Publishing/{project_id}/{project_id}_publish.mp4" "Youtube/Output/7. Publishing/{project_id}/published/"
mv "Youtube/Output/7. Publishing/{project_id}/{project_id}_metadata.yaml" "Youtube/Output/7. Publishing/{project_id}/published/"
```

Then remove the Desktop symlinks for this project (source folders remain intact):
```bash
bash "Youtube/Input/5. Tools/7. publishing_tools/remove_desktop_symlinks.sh" "{project_id}"
```

**Log:**
```
[moved] 0511 (1).mov  → published/
[moved] 0511 (1).yaml → published/
```

## YAML format reference

Required fields: `title`, `description`, `category_id`, `made_for_kids`

```yaml
title: "Your video title (max 100 characters)"

description: |
  Opening hook — first 150 chars appear in search results.

  📌 Timestamps
  0:00 Intro
  [add your timestamps]

  🔗 Links
  Join the community: [Skool link]

  #Hashtag1 #Hashtag2 #Hashtag3

category_id: "28"
made_for_kids: false
```

Optional fields (all have sensible defaults):
```yaml
category_id: "28"      # Science & Technology — default for this channel
thumbnail: "thumb.jpg" # relative to Publishing folder; omit if none
made_for_kids: false   # always false for this channel
```

**Do not include a `tags` or `privacy` field** — all uploads are scheduled by the publishing script.
