## Beat 4 — Demo: YouTube Scheduling

**Visual type:** screen-recording (phone)
**Camera:** iPhone screen recording is the primary visual. Face PIP remains visible in corner.
**Content:** Full live demo sequence from phone — three voice prompts with Codex executing between each:

  **Prompt 1:** Daniel says "Hey, can you tell me which videos are currently ready to be posted?" → Codex responds with a list of available videos on screen.

  **Prompt 2:** Daniel picks a video. Says "Is the description ready? Can you give me a quick summary — the video description and timestamps?" → Codex responds with description and timestamps displayed on phone screen. Hold on the response so it's readable.

  **Prompt 3:** Daniel says "Okay — go ahead and schedule this one for Friday at 4pm." → Codex executes. YouTube Studio loads on desktop (or visible via Codex), video appears as scheduled with description and timestamps filled in.

  Final hold: YouTube Studio showing the fully scheduled video — description written, timestamps in, publish date set. Hold several seconds.

**On-screen text (if any):** n/a — live screen content only
**Style notes:** Phone screen must be legible — record at high resolution. Codex chat interface should be clear and readable. If phone screen is too small in frame, consider screen mirroring to the desktop and capturing that instead.
**Timing notes:** Go quiet while Codex is executing each prompt — let the screen do the work. Only narrate once the result is visible. The "Three voice prompts. I never touched the computer." line lands on the final YouTube Studio hold.
