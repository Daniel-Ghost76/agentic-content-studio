# 03b — Workflow Map

**Sub-library:** `03_full_screen_overlays`  
**Status:** draft

## What It Is

A full-frame diagram showing a process, system, loop, or architecture. Nodes and connecting arrows build in step by step. Used when footage alone cannot show the system Daniel is describing.

## When To Use

- Explaining a multi-step workflow or automation pipeline
- Showing how systems connect to each other
- Before/after comparisons laid out as two parallel columns
- Mapping out a framework with inputs, steps, and outputs

## When Not To Use

- When the workflow has more than 6–7 nodes (too dense)
- When a simple numbered list is enough (use `02c_step_list`)
- When the diagram needs real-time data or dynamic content

## What Good Looks Like

- Full 1920×1080, `--bg` background
- Nodes: glass pill or rounded rect, label inside
- Arrows: thin lines with arrowheads, animate along their path as each step appears
- Build direction: left-to-right or top-to-bottom, one node at a time
- Optional title in top-left or top-centre
- Clean, readable at 1080p — no tiny labels
