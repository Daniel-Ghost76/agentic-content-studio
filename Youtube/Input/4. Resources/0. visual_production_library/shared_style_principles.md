# Shared Style Principles

These principles apply across all sub-libraries unless Daniel overrides them.

## Taste

- Clear beats clever.
- One visual idea per asset.
- Text must be readable at real viewing size.
- Motion should support comprehension, not show off.
- The visual should feel like Daniel's channel: useful, direct, premium enough, never generic.

## Avoid

- Busy dashboards where every tiny detail matters.
- Harsh neon, glitch spam, explosions, or visual noise.
- Long sentences inside visual overlays.
- Generic AI glow with no specific idea.
- Fake proof screens that look like real evidence unless clearly illustrative.

## Prompt Discipline

Prompts should include:

- intended use
- aspect ratio and duration, when relevant
- visual style
- exact text, if any
- composition and motion order
- anti-list
- final-frame requirement, when relevant

## Testing Discipline

Do not call a prompt reusable because it worked once. A reusable capsule needs repeatability: three reviewed outputs plus one fresh-session test.
