# Prompt

**Status:** draft

```text
Create a pre-recording proof/result slide.

Format: portrait or 16:9 based on the brief.

The slide should make one result obvious: something is scheduled, finished, generated, published, connected, or working. The proof/result screen should be the hero. Use short supporting text only.

Composition:
- short headline
- visible proof/result screen or output artifact
- optional status badge/checkmark
- one-line takeaway

Style:
clear, credible, modern, restrained, readable, evidence-led.

Avoid:
fake-looking proof, too much glow, tiny unreadable UI, long explanations, unrelated decorative devices.

Variables:
Headline: {headline}
Proof/result: {proof_description}
Takeaway: {takeaway}
```
