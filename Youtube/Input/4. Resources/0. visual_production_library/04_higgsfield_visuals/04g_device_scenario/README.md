# 04g — Device Scenario

**Sub-library:** `04_higgsfield_visuals`  
**Status:** draft — seeded from eRS3CmvrOvA (Nate Herk)

## What It Is

A Higgsfield clip of a phone or tablet device mockup with a teal glow outline, on a near-black background, showing a realistic scenario conversation or UI inside the device screen. Used to illustrate a concrete business use-case (e.g. a client problem, a workflow trigger) without using real footage.

## When To Use

- Daniel is describing a client scenario: "imagine a real estate agency tells you…"
- Illustrating what a trigger event looks like (a message, a form, a notification)
- Any "here's the problem they're facing" moment where a phone/app context makes it tangible

## When Not To Use

- When a laptop/desktop UI is more appropriate (use `04m_software_demo_mockup`)
- For abstract concepts with no device context (use `04c_abstract_concept`)
- When a real screen recording is available and more accurate

## What Good Looks Like

- Device: modern phone (iPhone-style), portrait orientation, occupying roughly centre-to-left of frame
- Device outline: teal glow, `2–3px`, `rgba(16,185,129,0.85)`, possibly animated pulse
- Screen content: a realistic-looking app UI — could be a messaging thread, a CRM, a booking system — showing the scenario text
- Background: near-black, subtle teal ambient glow behind the device
- Duration: 3–5 seconds, mostly static with gentle glow pulse or subtle device float
