# 04b — Workflow Transformation

**Sub-library:** `04_higgsfield_visuals`  
**Status:** draft

## What It Is

A Higgsfield clip depicting a transformation — something changing state, evolving, or moving from one phase to another. Used for before/after moments, system-in-motion concepts, or "things are changing" narrative beats.

## When To Use

- Transitioning from problem to solution in the narrative
- Showing a process "in action" (agents working, pipeline running)
- A before/after section where the abstract transformation needs to be felt visually

## When Not To Use

- When a static diagram is clearer (use `03b_workflow_map`)
- When a literal screen recording is available
- For purely decorative moments with no narrative purpose

## What Good Looks Like

- Clear sense of directional motion (left→right, bottom→top, dark→light)
- Abstract enough to be reusable across different transformation contexts
- Premium, restrained — not explosive or chaotic
- Final frame should feel resolved, not mid-motion
