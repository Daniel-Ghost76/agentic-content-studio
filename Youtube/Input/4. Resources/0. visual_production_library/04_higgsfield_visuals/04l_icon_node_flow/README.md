# 04l — Icon Node Flow

**Sub-library:** `04_higgsfield_visuals`  
**Status:** draft — seeded from eRS3CmvrOvA (Nate Herk)

## What It Is

A Higgsfield clip showing 3–4 large icon circles (teal glow rings) connected by simple arrows, with a label beneath each circle. Used to explain a multi-step retrieval or process mechanism — particularly when the steps are conceptual (index → timeline → details) rather than literal software steps.

## When To Use

- Explaining how a system retrieves or processes information in stages
- Any "it first does X, then Y, then Z" moment where each stage has a distinct icon
- Making an abstract multi-stage process visual and digestible

## When Not To Use

- When steps are software-literal (use `04k_linear_workflow_steps` with a window frame)
- When there are only 2 nodes — too sparse; use a single diagram element or `04j`
- For non-sequential components (use `04j_architecture_diagram`)

## What Good Looks Like

- Background: near-black, deep teal gradient glow in corners or base
- Nodes: large circles (~180–220px diameter), teal glow ring outline (not filled), icon inside (~80px)
- Connections: simple thin arrows between circles (→), white or teal
- Labels: beneath each circle, white text, `20–24px`, 1–3 words
- Layout: horizontal spread across the frame, nodes equally spaced
- Animation: nodes pulse gently with the glow, arrows draw in sequentially
- Duration: 4–6 seconds
