## Prompt Template

```
MacBook laptop product shot on near-black background (#09090b). Soft teal ambient 
glow behind and beneath the laptop. The laptop is slightly angled (~10° perspective 
tilt), screen open and illuminated.

The laptop screen shows: {screen_content_description} — a realistic dark-mode 
software interface (Claude Code in VSCode, a terminal, or a browser app). 
The screen should look active and in-use.

Overlaid on the top-center of the laptop screen: a pill badge "{badge_text}" — 
a small rounded pill with a dark teal background and white text, possibly with 
a checkmark "✓" prefix. This badge identifies the skill or feature being shown.

The laptop rests on a subtle surface with a faint reflection below.
Minimal idle animation: very slight screen glow pulse.

Aspect ratio: 16:9. Duration: {duration_s} seconds.
```

---

## Variables

- `{screen_content_description}` — what the laptop screen shows (e.g. `"Claude Code chat interface in VS Code, showing a conversation with a skill suggestion active"`)
- `{badge_text}` — text on the overlay badge pill (e.g. `"✓ Superpowers"`, `"✓ GSD Active"`, `"Context Mode ON"`)
- `{duration_s}` — clip duration in seconds (default: `5`, range: `4–7`)

## Hybrid Approach Note

For maximum quality and text accuracy, consider a two-pass approach:
1. Generate the **laptop frame + background + glow** in Higgsfield (no screen content, black screen)
2. Composite the **actual screen recording or screenshot** into the laptop screen using HyperFrames perspective transform
3. Composite the **badge pill** using `02f_icon_pill` or `02e_version_badge` overlaid on top

This gives full control over screen content and badge text without relying on Higgsfield's text generation.

---

## Anti-list

- No hands or person in the shot — laptop only
- Screen must show dark-mode UI — no bright white interfaces
- Badge pill must be small and unobtrusive — top-center, not blocking the main content
- No animated typing or scrolling generated by Higgsfield — static screen content only
