# Visuals SOP (Stage 5)

Codex executes this stage.

The visuals stage has two SOPs that run in sequence. Stage 5 takes both the graded/zoomed base video and the approved overlay map from Stage 4.

## Pipeline

```
From Stage 4:
  {project_id}_cut_final.mp4   ← graded + zoom-baked base video
  {project_id}_overlay_map.md  ← all overlay stubs (approved by Daniel)
    ↓
5a. Overlays  → builds all overlays from the map, composites everything
    ↓
5b. Finish    → adds intro/outro + music (optional) + runs auto QC
    ↓
Stage 6 (Review) receives: {project_id}_final.mp4 + {project_id}_qc_report.md
```

## SOPs

| Stage | File | Executor | Job |
|-------|------|----------|-----|
| 5a | `5a. overlays_SOP.md` | Codex | Screen rec cleanup + build all overlays (HyperFrames, Higgsfield) + composite |
| 5b | `5b. finish_SOP.md` | Codex | Branded intro/outro + optional music + automatic QC |

## Relationship to Stage 3

Stage 3 (Pre-production Materials) creates slides, diagrams, and static assets Daniel records with. Stage 5 creates post-recording overlays and composited motion graphics. These are different stages with different inputs and outputs. Do not confuse them.

## Visual Library Rule

Stage 5 outputs must record the Visual Production Library capsule used for each visual asset. If the approved overlay map names no capsule, the Stage 5 agent chooses from the relevant sub-library or asks Daniel before creating a new draft capsule.
