# Analytics SOP

Claude executes this stage by default.

Pull performance snapshots for published videos and generate monthly channel rollups. Runs silently after the one-time Google Cloud setup.

## Delegation (token-saving)
Delegate the data collection to the `analyst` subagent (Haiku, runs off your context). It runs `check_analytics_due.py`, `fetch_analytics.py`, and `assemble_monthly_rollup.py`, returning only the due-verdict, a metric summary, and underperformer flags — no raw API payloads in your context. Interpret the signal and feed it back to Ideation yourself.

---

## Role

The Analytics SOP is responsible for measuring video performance after publishing. It owns Stage 9 — from reading the upload log to generating snapshots and monthly rollups. It turns raw numbers into a clear signal that feeds back into Ideation (Stage 1).

---

## Trigger

Activate this SOP when:
- The user runs `/analytics` in Claude Code
- A video's Day 7 or Day 30 window has opened (based on publish date in `upload_log.md`)
- It's the last week of the month (monthly rollup)

## Before first run — enable YouTube Analytics API

The YouTube Analytics API must be enabled in the same Google Cloud project as the YouTube Data API.

If you haven't done this yet:
1. Go to console.cloud.google.com → your project
2. **APIs & Services → Library** → search **YouTube Analytics API** → **Enable**

The next run will open a browser to re-auth with the expanded permissions. After that, all future runs are silent.

## Step 1 — Read the upload log

Read `Youtube/Output/9. Analytics/upload_log.md` and parse every row.

For each row, extract:
- `title` — video title
- `project_id` — canonical project ID
- `video_id` — YouTube video ID
- `scheduled_for` — the date the video went live (format: `YYYY-MM-DD HH:MM London`)
  - Extract the date part only: `YYYY-MM-DD`

If the log is empty or has no rows, print:
```
No published videos found in upload_log.md. Publish a video first with /publish.
```
Then stop.

---

## Step 2 — Determine which snapshots are due

Run this script — do not compute dates manually:
```bash
python3 "Youtube/Input/5. Tools/9. analytics_tools/check_analytics_due.py"
```
Read its output table. Fetch only rows marked `FETCH`. If it prints "All snapshots current — nothing to fetch", jump to Step 4.

---

## Step 3 — Fetch missing snapshots

For each missing snapshot, run:

```bash
  "Youtube/Input/4. Resources/7. publishing_resources/venv/bin/python3" \
  "Youtube/Input/4. Resources/7. publishing_resources/fetch_analytics.py" \
  --project-id "<PROJECT_ID>" \
  --video-id "<VIDEO_ID>" \
  --title "<TITLE>" \
  --published "<YYYY-MM-DD>" \
  --snapshot d07|d30 \
  --output-dir "Youtube/Output/9. Analytics"
```

Parse the output for `SNAPSHOT_SAVED=` to confirm success.

**Log after each fetch:**
```
[fetch] 02_codex_mobile d07 → saved to Youtube/Output/9. Analytics/02_codex_mobile/02_codex_mobile_analytics_d07.md
```

If the script exits with an error (data not yet available), log it and skip:
```
[skip] I_8GpWlf6oc d07 → analytics data not yet available (YouTube has a 2-3 day delay)
```

---

## Step 4 — Monthly rollup (last week of month only)

Check today's date. If the day of the month is 24 or later, run:
```bash
python3 "Youtube/Input/5. Tools/9. analytics_tools/assemble_monthly_rollup.py" --month YYYY-MM
```
Replace `YYYY-MM` with the current month. The script skips automatically if the rollup already exists (Rule R7). If it prints that the rollup was saved, confirm the path to Daniel.

The rollup format it produces:

```markdown
# May 2026 — Monthly Channel Summary

Videos published this month: 1
Total views: 1,234
Total watch time: 6,417 min
Net subscribers: +34

## Video Rankings

### By avg view duration %
1. "Day 1 of Building an AI Business from Scratch" — 52% (d07)

### By CTR
1. "Day 1 of Building an AI Business from Scratch" — 13.9% (d07)

### By subscribers gained
1. "Day 1 of Building an AI Business from Scratch" — 34 subs (d07)
```

Save to `Youtube/Output/9. Analytics/YYYY-MM-monthly.md`.

**Log:**
```
[rollup] Generated 2026-05-monthly.md
```

If it's not the last week of the month, skip with:
```
[rollup] Monthly rollup not due until last week of May
```
