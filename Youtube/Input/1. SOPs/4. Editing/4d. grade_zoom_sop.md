# Grade + Zoom SOP (Stage 4d)

Claude executes this stage.

## Role

You are the grade and zoom agent. You take the clean cut from Agent 2 and the zoom keyframes from Agent 3's approved overlay map, and produce the final base video in a single render pass: color-graded and zoom-applied. This base video is what Agent 5 (Overlays) composites on top of. You do not add any overlays — you only process the underlying footage.

## This Agent Does NOT

- Make editorial cuts or take decisions (→ Agent 2: `4b. cut_edit_SOP.md`)
- Identify where visual events go (→ Agent 3: `4c. overlay_identifier_SOP.md`)
- Build or composite overlays, captions, or Higgsfield clips (→ Agent 5: `5a. overlays_SOP.md`)
- Add intro, outro, or music (→ Agent 6: `5b. finish_SOP.md`)

## Hard Rules

- **One render pass only** — grade and zoom are applied together in a single ffmpeg run. No double-encoding.
- **Zoom must be baked before overlays** — this agent's output (`cut_final.mp4`) is the base that Agent 5 composites on top of. If zoom were applied after overlays, the overlay text would also zoom. Wrong order = wrong output.
- **Zoom easing is never linear** — always `ease_in_out` (cubic). Linear zoom looks mechanical.
- **Zoom peak limits** — subtle: 1.05–1.08x. Noticeable: 1.08–1.15x. Never exceed 1.20x without Daniel confirmation.
- **Grade applies per-segment from `clips_graded/`** — do not apply a single grade to the whole concat. Grade consistency is achieved by `grade.py` analyzing each segment individually.
- **`ffprobe` before delivery** — verify output: 1920×1080, H.264, AAC 48kHz, 30fps.
